## 2. 方案一：Redis 工具缓存

### 2.1 设计目标

在现有 ConcurrentHashMap 内存缓存基础上，增加 Redis 二级缓存：
- **L1 缓存**（内存）：McpClientManager 的 ConcurrentHashMap，极速访问
- **L2 缓存**（Redis）：工具定义 JSON 序列化存储，跨实例共享、重启不丢失

```
工具调用请求
    │
    ▼
L1 缓存 (ConcurrentHashMap) ──Hit──→ 返回工具定义
    │ Miss
    ▼
L2 缓存 (Redis) ──Hit──→ 回填 L1 → 返回工具定义
    │ Miss
    ▼
MySQL 查询 ──→ 回填 L2 + L1 → 返回工具定义
```

### 2.2 实现方案

```java
package com.example.agent.application.tool;

import com.example.agent.domain.tool.ToolRegistry;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.stereotype.Component;

import java.time.Duration;
import java.util.Optional;

/**
 * 工具定义二级缓存（L2: Redis）.
 *
 * Key 格式: tool:def:{tenantId}:{toolId}
 * TTL: 1 小时（比 L1 的 5min 刷新周期更长，因为 Redis 可跨实例共享）
 */
@Slf4j
@Component
@RequiredArgsConstructor
public class ToolCacheManager {

    private final RedisTemplate<String, ToolRegistry> redisTemplate;
    private static final String KEY_PREFIX = "tool:def:";
    private static final Duration TTL = Duration.ofHours(1);

    public Optional<ToolRegistry> get(String tenantId, String toolId) {
        String key = KEY_PREFIX + tenantId + ":" + toolId;
        ToolRegistry tool = redisTemplate.opsForValue().get(key);
        if (tool != null) {
            log.debug("Redis cache HIT for tool: {}", toolId);
            return Optional.of(tool);
        }
        log.debug("Redis cache MISS for tool: {}", toolId);
        return Optional.empty();
    }

    public void put(ToolRegistry tool) {
        String key = KEY_PREFIX + tool.getTenantId() + ":" + tool.getToolId();
        redisTemplate.opsForValue().set(key, tool, TTL);
    }

    public void evict(String tenantId, String toolId) {
        String key = KEY_PREFIX + tenantId + ":" + toolId;
        redisTemplate.delete(key);
        log.info("Redis cache EVICT for tool: {}", toolId);
    }

    /** 工具启停或配置变更时，清除 L1+L2 缓存 */
    public void refresh(String tenantId, String toolId) {
        evict(tenantId, toolId);
        // McpClientManager 下次 get() 时 L1 miss → L2 miss → MySQL → 回填
    }
}
```

### 2.3 集成到 ToolApplicationService

```java
// 修改 ToolApplicationService.java

public ToolResponse getTool(String toolId) {

}
```

## 3. 方案二：工具版本化

### 3.1 设计目标

工具配置变更时保留历史版本，支持回滚。类似提示词模板的版本管理模式（Memento 模式）。

### 3.2 数据模型

```sql
-- V1.5.0: 工具版本化
CREATE TABLE t_tool_registry_version (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,
    tenant_id       VARCHAR(64)  NOT NULL,
    tool_id         VARCHAR(64)  NOT NULL COMMENT '关联 t_tool_registry.tool_id',
    version         INT          NOT NULL DEFAULT 1,
    tool_name       VARCHAR(128) NOT NULL,
    tool_type       VARCHAR(20)  NOT NULL,
    endpoint_url    VARCHAR(512),
    input_schema    JSON,
    output_schema   JSON,
    auth_config     JSON,
    require_approval TINYINT(1)  DEFAULT 0,
    description     VARCHAR(512),
    change_reason   VARCHAR(256) COMMENT '变更原因',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_tool_version (tenant_id, tool_id, version DESC)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.3 版本生命周期

```
创建工具(v1) → 编辑 → 保存为 v2
                         │
                         ├─ v1 写入 t_tool_registry_version
                         └─ v2 覆盖 t_tool_registry

运行时调用: 始终使用 t_tool_registry 的当前版本 (latest)

需要回滚时:
  POST /tools/{id}/rollback?version=1
  → 从 t_tool_registry_version 读取 v1
  → 覆盖 t_tool_registry 当前字段
  → 当前版本成为 v3（v1 的内容 + 新版本号）
```

### 3.4 实现方案

```java
// ToolDomainService.java 新增方法

/**
 * 更新工具配置，自动创建版本快照.
 */
public ToolRegistry updateWithVersion(String toolId, UpdateToolRequest request, String changeReason) {
    ToolRegistry current = repository.findByToolId(toolId)
        .orElseThrow(() -> new ResourceNotFoundException("工具", toolId));

    // 1. 快照当前版本 → t_tool_registry_version
    ToolRegistryVersion snapshot = ToolRegistryVersion.from(current, changeReason);
    versionRepository.save(snapshot);

    // 2. 应用更新 → version +1
    current.applyUpdate(request);
    current.incrementVersion();
    repository.save(current);

    // 3. 清除缓存
    toolCacheManager.refresh(current.getTenantId(), toolId);

    return current;
}

/**
 * 回滚到指定版本.
 */
public ToolRegistry rollback(String toolId, int targetVersion) {
    ToolRegistry current = repository.findByToolId(toolId)
        .orElseThrow(() -> new ResourceNotFoundException("工具", toolId));

    // 1. 从版本历史读取目标版本
    ToolRegistryVersion target = versionRepository
        .findByToolIdAndVersion(toolId, targetVersion)
        .orElseThrow(() -> new ResourceNotFoundException("工具版本", targetVersion));

    // 2. 快照当前版本（保留回滚前的状态）
    ToolRegistryVersion preRollbackSnapshot = ToolRegistryVersion.from(
        current, "回滚到 v" + targetVersion + " 前的自动快照");
    versionRepository.save(preRollbackSnapshot);

    // 3. 应用目标版本的内容
    current.restoreFromVersion(target);
    current.incrementVersion();
    repository.save(current);

    // 4. 清除缓存
    toolCacheManager.refresh(current.getTenantId(), toolId);

    return current;
}
```

## 4. 方案三：MCP 心跳检测

### 4.1 设计目标

在现有 `@Scheduled(fixedDelay=5min)` 定时刷新的基础上，增加**主动心跳检测**：
- 对 ACTIVE 状态的 MCP 工具，定期发送 `ping` / `list_tools` 请求
- 连续失败 N 次后自动标记为 DISABLED
- 恢复后自动标记为 ACTIVE

### 4.2 实现方案

```java
package com.example.agent.infrastructure.mcp;

import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.scheduling.annotation.Scheduled;
import org.springframework.stereotype.Component;

import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;

/**
 * MCP 心跳检测器.
 *
 * 每 30 秒对所有 ACTIVE 的 MCP 工具发送 list_tools 探测请求.
 * 连续失败 3 次 → 自动 DISABLED + 告警日志.
 * 恢复探测成功 1 次 → 自动恢复 ACTIVE + 通知.
 */
@Slf4j
@Component
@RequiredArgsConstructor
public class McpHeartbeatDetector {

    private final ToolRegistryRepository toolRepository;
    private final ToolDomainService toolDomainService;
    private final McpClientManager mcpClientManager;

    /** 连续失败计数器: toolId → failureCount */
    private final Map<String, Integer> failureCounters = new ConcurrentHashMap<>();

    private static final int MAX_FAILURES = 3;       // 连续失败阈值
    private static final int HEARTBEAT_INTERVAL = 30_000;  // 30 秒

    @Scheduled(fixedDelay = HEARTBEAT_INTERVAL)
    public void detectHeartbeat() {
        List<ToolRegistry> activeMcpTools = toolRepository.findByTypeAndStatus(
            ToolType.MCP, ToolStatus.ACTIVE);

        for (ToolRegistry tool : activeMcpTools) {
            try {
                // 发送 list_tools 探测请求（轻量级，仅验证连通性）
                boolean alive = mcpClientManager.ping(tool.getToolId());

                if (alive) {
                    // 成功 → 重置计数器
                    Integer previousFailures = failureCounters.remove(tool.getToolId());
                    if (previousFailures != null && previousFailures >= MAX_FAILURES) {
                        log.warn("MCP 工具已恢复: {} (之前已自动禁用)", tool.getToolId());
                        // 如果之前因心跳失败被禁用，这里不做自动恢复（需人工确认）
                    }
                } else {
                    recordFailure(tool);
                }
            } catch (Exception e) {
                log.error("MCP 心跳检测异常: toolId={}, error={}",
                    tool.getToolId(), e.getMessage());
                recordFailure(tool);
            }
        }
    }

    private void recordFailure(ToolRegistry tool) {
        int failures = failureCounters.merge(tool.getToolId(), 1, Integer::sum);
        log.warn("MCP 心跳失败: toolId={}, 连续失败次数={}/{}",
            tool.getToolId(), failures, MAX_FAILURES);

        if (failures >= MAX_FAILURES) {
            // 自动禁用
            toolDomainService.disable(tool.getToolId());
            // 清除 MCP Client 连接缓存
            mcpClientManager.removeClient(tool.getToolId());

            log.error("MCP 工具已自动禁用: toolId={}, 原因: 连续 {} 次心跳失败",
                tool.getToolId(), MAX_FAILURES);
            // TODO: 发送告警通知（飞书/钉钉/Telegram）
        }
    }
}
```

### 4.3 与现有 @Scheduled 刷新的关系

```
现有: @Scheduled(fixedDelay=5min) McpClientManager.refreshClients()
        → 重新从 DB 加载所有 ACTIVE 工具 → 重建 MCP 连接

新增: @Scheduled(fixedDelay=30s) McpHeartbeatDetector.detectHeartbeat()
        → 对 ACTIVE 工具发送轻量 ping → 快速发现失联

两者互补：
- 心跳检测（30s）：快速发现故障 → 自动禁用
- 定时刷新（5min）：全量重建连接 → 发现新工具
```
