## 二、当前系统差距分析

### 已存在的（可直接复用）

| 组件 | 文件 | 状态 |
|------|------|------|
| KB CRUD | `KnowledgeBaseApplicationService.java` | 基本可用，需扩展 |
| 文档上传/下载 | `DocumentApplicationService.java` | 可用，需改上传流程 |
| 异步解析管线 | `DocumentPipelineOrchestrator.java` | 可用，需改触发方式 |
| 混合检索 | `HybridSearchApplicationService.java` | **有 Bug**：knowledgeId 参数未实际过滤 |
| 检索结果溯源 | `SearchResultDTO.java` (HitItem.doc*) | **部分实现**：已定义字段，需确保全路径填充 |
| MinIO 存储 | `DocumentApplicationService` (MinioClient) | 可用 |
| Milvus 向量 | `MilvusStoreClient.java` (端口) | 缺少 delete 方法 |
| RBAC 权限 | Sa-Token 注解 | 可用，需加新权限码 |
| 租户隔离 | `TenantContext` | 可用 |

### 核心差距（必须填补）

1. **KB 无 `createdBy`** → 无法实现需求 7（创建者权限控制）
2. **KB 状态是裸 String "ACTIVE"** → 需要 `KnowledgeBaseStatus` 枚举（ENABLED/DISABLED/DELETED）
3. **文档上传自动触发解析** → 需改为手动触发（`PENDING_PARSE` 初始状态）
4. **`DocumentStatus` 枚举混淆处理状态与生命周期状态** → 需要新状态值
5. **`MilvusStoreClient` 无 `delete` 方法** → 需求 4/5 无法实现
6. **`HybridSearchApplicationService.search()` 收到 `knowledgeId` 参数但不使用** → 需求 6 的核心 Bug
7. **`VectorEntry` 无 `knowledgeId` 字段** → Milvus 中无法按 KB 过滤
8. **删除 KB 不会级联处理文档** → 需求 5 未实现

## 三、领域模型变更

### 3.1 新增 `KnowledgeBaseStatus` 枚举

**文件**: `domain/knowledge/valueobject/KnowledgeBaseStatus.java`（新建）

```
ENABLED  → 正常：可上传、可检索、可解析
DISABLED → 弃用：不可上传、不可检索、数据保留、可恢复
DELETED  → 删除：级联清理、终态、不可逆
```

### 3.2 替换 `DocumentStatus` 枚举

**文件**: `domain/knowledge/valueobject/DocumentStatus.java`（重写）

旧值 → 新值映射：
```
UPLOADING  → PENDING_PARSE   （上传完成，等待手动触发解析）
PARSING    → PARSING         （异步解析中，不变）
CHUNKING   → CHUNKING        （异步切分中，不变）
EMBEDDING  → EMBEDDING       （异步向量化中，不变）
READY      → PARSED          （已解析，向量可用）
FAILED     → FAILED          （处理失败，可重试）
[新增]     → DEPRECATED      （已弃用，向量已删除，元数据保留）
```

状态机转换图：
```
uploadFile() → PENDING_PARSE
                  │ triggerParse()
                  v
              PARSING → CHUNKING → EMBEDDING → PARSED
                  │         │          │
                  └─────────┼──────────┘
                            │ error
                            v
                         FAILED ──→ retry → PENDING_PARSE
                         
PARSED/FAILED ──deprecate()──→ DEPRECATED （删除向量）
PENDING_PARSE/FAILED/DEPRECATED ──delete()──→ [软删除]
```

### 3.3 `KnowledgeBase` 实体扩展


### 3.7 新增 `DocumentLifecycleDomainService`

**文件**: `domain/knowledge/service/DocumentLifecycleDomainService.java`（新建）

状态转换校验：
- `assertCanParse(doc)` — 仅 PENDING_PARSE / FAILED 可解析
- `assertCanDeprecate(doc)` — 仅 PARSED / FAILED 可弃用
- `assertCanDelete(doc)` — 仅 PENDING_PARSE / FAILED / DEPRECATED 可删除（PARSED 必须先弃用）

### 3.8 `KnowledgeBaseDomainService` 扩展

**文件**: `domain/knowledge/service/KnowledgeBaseDomainService.java`（修改）

新增校验：
- `assertKbEnabled(kb)` — 仅 ENABLED 可上传/解析
- `assertKbNotDeleted(kb)`
- `assertCreatorAccess(kb, userId)` — 需求 7：仅创建者可删除
- `assertCanDisable(kb)` / `assertCanEnable(kb)`
- `assertCanDelete(kb)` — KB 必须先 DISABLED 再 DELETED

### 需求 7 实现规则

```
文档删除权限 = 当前用户 == KB.createdBy OR 当前用户 has "kb:admin"
KB 删除权限   = 当前用户 == KB.createdBy OR 当前用户 has "kb:admin"
所有租户成员 → 可检索、可上传、可查看
非创建者       → 不可删除（返回 403）
```

## 七、状态机

### KB 状态
| 状态 | 上传 | 检索 | 解析 | 下载 | 可逆 |
|------|------|------|------|------|------|
| ENABLED | ✅ | ✅ | ✅ | ✅ | → DISABLED |
| DISABLED | ❌ | ❌ | ❌ | ✅ | → ENABLED |
| DELETED | ❌ | ❌ | ❌ | ❌ | ❌ 终态 |

### 文档状态
| 状态 | 中文标签 | 向量存在 | 可解析 | 可弃用 | 可删除 |
|------|----------|----------|--------|--------|--------|
| PENDING_PARSE | 待解析 | ❌ | ✅ | ❌ | ✅ |
| PARSING | 解析中 | ❌ | ❌ | ❌ | ❌ |
| CHUNKING | 切分中 | ❌ | ❌ | ❌ | ❌ |
| EMBEDDING | 向量化中 | 部分 | ❌ | ❌ | ❌ |
| PARSED | 已解析 | ✅ | ❌ | ✅ | ❌ |
| DEPRECATED | 已弃用 | ❌ | ❌ | ❌ | ✅ |
| FAILED | 失败 | ❌ | ✅(重试) | ✅ | ✅ |

## 十、风险与注意事项

| 风险 | 缓解措施 |
|------|----------|
| Milvus 旧 Collection 无 `knowledge_id` 字段 | 提供重建脚本；代码 fallback 到无过滤模式 |

| 全文检索返回弃用文档 | SQL 加 `dc.deleted=0 AND d.status NOT IN ('DEPRECATED','DELETED')` |

## 十一、验证方案

1. **单元测试**: 状态机转换合法性（`DocumentLifecycleDomainService` 全组合覆盖）
2. **集成测试**: 创建 KB → 上传 → 解析 → 检索；弃用 → 向量删除验证；级联删除全链路
3. **回归测试**: 已有 KB CRUD + 文档 CRUD + 检索 API 不受影响
4. **端到端**: PDF 上传 → 待解析 → 触发解析 → PARSED → 指定 KB 检索 → 命中 → 下载链接可用
5. **需求 8**: 检索结果中 `documentFilename` + `documentAccessUrl` 非空
