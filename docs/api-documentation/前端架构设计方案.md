## 一、项目概述

### 1.1 背景

Agent Platform 是一个企业级 AI 代理平台，提供以下核心能力：

- **对话管理**: 流式聊天、会话历史、消息反馈
- **知识库引擎**: RAG检索、文档管理、混合搜索
- **任务规划**: DAG任务编排、工具调用、执行监控
- **权限体系**: 多租户、RBAC角色权限、审批工作流
- **安全治理**: 敏感词过滤、安全事件审计
- **效果评估**: BadCase闭环、优化工单、数据集评测
- **可观测性**: 全链路追踪、日志分析、性能监控

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| **前后端分离** | 纯静态资源部署，通过 API 与后端交互 |
| **配置化驱动** | 请求域名、超时时间、功能开关全部可配置 |
| **模块化架构** | 按业务域拆分模块，支持按需加载和扩展 |
| **类型安全** | TypeScript 全量覆盖，API 接口自动生成类型定义 |
| **响应式设计** | 适配桌面端和移动端（优先桌面端） |
| **实时通信** | WebSocket + SSE 双通道支持流式对话和实时推送 |

### 2.2 为什么不选择 Redux/MobX？

- **Zustand 优势**: 
  - 更简洁的 API（基于 Hooks）
  - 无样板代码，直接操作 state
  - 天然支持 TypeScript
  - 体积更小（~1KB）
  - 适合中小型项目

```typescript
// src/config/app.config.ts
interface AppConfig {
  api: {
    baseURL: string;
    timeout: number;
    retryCount: number;
    retryDelay: number;
  };
  websocket: {
    url: string;
    heartbeatInterval: number;
    reconnectAttempts: number;
    reconnectDelay: number[];
  };
  features: Record<string, boolean>;
  chat: {
    maxMessageLength: number;
    pageSize: number;
    streamTimeout: number;
  };
  upload: {
    maxFileSize: number;
    allowedTypes: string[];
  };
}

let appConfig: AppConfig | null = null;

export async function loadAppConfig(): Promise<AppConfig> {
  if (appConfig) return appConfig;
  
  const response = await fetch('/config.json');
  appConfig = await response.json();
  return appConfig;
}

export function getAppConfig(): AppConfig {
  if (!appConfig) {
    throw new Error('App config not loaded. Call loadAppConfig() first.');
  }
  return appConfig;
}
```

**优势**:
- 无需重新构建即可修改配置
- 支持不同客户定制化配置

**API 路径集中管理**:

所有 Service 文件中的 API 路径不得硬编码 `/api/v1/` 前缀，必须从 `constants.ts` 导入：

```typescript

/** 聊天配置 — 从 config.json 的 chat 段提取的默认值 */
export const CHAT_CONFIG = {
  MAX_MESSAGE_LENGTH: 4000,
  PAGE_SIZE: 50,
  STREAM_TIMEOUT: 300_000,

export const UPLOAD_CONFIG = {
  MAX_FILE_SIZE: 52_428_800,  // 50MB
  ALLOWED_TYPES: ['pdf', 'docx', 'txt', 'md'],

```

### 4.2 Axios 实例配置

```typescript
// src/services/apiClient.ts
import axios, { AxiosInstance, AxiosError } from 'axios';
import { getAppConfig } from '@/config/app.config';
import { useAuthStore } from '@/stores/useAuthStore';
import { message } from 'antd';

class ApiClient {
  private instance: AxiosInstance;

  constructor() {
    const config = getAppConfig();
    
    this.instance = axios.create({
      baseURL: config.api.baseURL,
      timeout: config.api.timeout,
      headers: {
        'Content-Type': 'application/json',
      },
    });

    // 请求拦截器
    this.instance.interceptors.request.use(
      (config) => {
        const token = useAuthStore.getState().token;
        if (token) {
          config.headers.Authorization = `Bearer ${token}`;
        }
        return config;
      },
      (error) => Promise.reject(error)
    );

    // 响应拦截器
    this.instance.interceptors.response.use(
      (response) => response.data,
      (error: AxiosError) => {
        if (error.response?.status === 401) {
          useAuthStore.getState().logout();
          window.location.href = '/login';
          message.error('登录已过期，请重新登录');
        } else if (error.response?.status === 403) {
          message.error('权限不足');
        } else if (error.response?.status >= 500) {
          message.error('服务器错误，请稍后重试');
        }
        return Promise.reject(error);
      }
    );
  }

  getInstance(): AxiosInstance {
    return this.instance;
  }
}

export const apiClient = new ApiClient().getInstance();
```

### 5.1 后端接口总览

根据扫描的后端 Controller，整理出以下 API 模块：

| 模块 | 基础路径 | 主要功能 | 权限要求 |
|------|---------|---------|---------|

```typescript
// src/hooks/useSSE.ts
import { useState, useCallback, useRef } from 'react';
import { getAppConfig } from '@/config/app.config';
import { useAuthStore } from '@/stores/useAuthStore';

interface SSEOptions {
  onMessage?: (data: any) => void;
  onError?: (error: Event) => void;
  onOpen?: () => void;
}

export function useSSE(options: SSEOptions = {}) {
  const [isLoading, setIsLoading] = useState(false);
  const eventSourceRef = useRef<EventSource | null>(null);
  const config = getAppConfig();
  const token = useAuthStore(state => state.token);

  const connect = useCallback((url: string, body: any) => {
    setIsLoading(true);
    
    // SSE 不支持 POST，需要通过特殊方式传递参数
    // 方案1: URL参数（注意长度限制）
    // 方案2: 先POST获取eventId，再GET订阅
    // 本项目采用方案2
    
    const eventSource = new EventSource(
      `${config.api.baseURL}${url}?token=${token}`,
      { withCredentials: true }
    );

    eventSource.onopen = () => {
      options.onOpen?.();
    };

    eventSource.onmessage = (event) => {
      try {
        const data = JSON.parse(event.data);
        options.onMessage?.(data);
        
        // 流式结束标记
        if (data.type === 'done') {
          eventSource.close();
          setIsLoading(false);
        }
      } catch (error) {
        console.error('SSE parse error:', error);
      }
    };

    eventSource.onerror = (error) => {
      options.onError?.(error);
      eventSource.close();
      setIsLoading(false);
    };

    eventSourceRef.current = eventSource;
  }, [token, options]);

  const disconnect = useCallback(() => {
    if (eventSourceRef.current) {
      eventSourceRef.current.close();
      eventSourceRef.current = null;
      setIsLoading(false);
    }
  }, []);

  return { isLoading, connect, disconnect };
}
```

后端 SSE 端点 `GET /api/v1/conversations/{id}/stream` 推送以下事件流：

```typescript
// src/types/sse.d.ts — SSE 事件类型联合

/** Token 用量统计 */
interface TokenUsage {
  inputTokens: number;
  outputTokens: number;
  totalTokens: number;
}

/** SSE 事件流联合类型 */

  | { type: 'token'; content: string }                          // 增量 Token
  | { type: 'tool_call'; toolName: string; args: any }          // 工具调用
  | { type: 'tool_result'; toolName: string; result: any }      // 工具结果
  | { type: 'thinking'; content: string }                       // 思考过程（折叠显示）

```

tool_result → 渲染工具返回摘要

approval_required → 触发审批卡片渲染

### 5.4 WebSocket 实时推送实现

```typescript
// src/hooks/useWebSocket.ts
import { useState, useEffect, useCallback, useRef } from 'react';
import { getAppConfig } from '@/config/app.config';
import { useAuthStore } from '@/stores/useAuthStore';

interface WebSocketMessage {
  type: string;
  payload: any;
  timestamp: number;
}

interface UseWebSocketOptions {
  onMessage?: (message: WebSocketMessage) => void;
  onOpen?: () => void;
  onClose?: () => void;
  onError?: (error: Event) => void;
}

export function useWebSocket(options: UseWebSocketOptions = {}) {
  const [isConnected, setIsConnected] = useState(false);
  const wsRef = useRef<WebSocket | null>(null);
  const reconnectTimerRef = useRef<NodeJS.Timeout | null>(null);
  const config = getAppConfig();
  const token = useAuthStore(state => state.token);
  const reconnectAttempts = useRef(0);

  const connect = useCallback(() => {
    if (!token) return;

    const wsUrl = `${config.websocket.url}?token=${token}`;
    const ws = new WebSocket(wsUrl);

    ws.onopen = () => {
      console.log('[WebSocket] Connected');
      setIsConnected(true);
      reconnectAttempts.current = 0;
      options.onOpen?.();

      // 启动心跳
      startHeartbeat(ws);
    };

    ws.onmessage = (event) => {
      try {
        const message: WebSocketMessage = JSON.parse(event.data);
        
        // 处理心跳响应
        if (message.type === 'pong') return;

        options.onMessage?.(message);
      } catch (error) {
        console.error('[WebSocket] Parse error:', error);
      }
    };

    ws.onclose = () => {
      console.log('[WebSocket] Closed');
      setIsConnected(false);
      options.onClose?.();
      
      // 自动重连
      scheduleReconnect();
    };

    ws.onerror = (error) => {
      console.error('[WebSocket] Error:', error);
      options.onError?.(error);
    };

    wsRef.current = ws;
  }, [token, options]);

  const disconnect = useCallback(() => {
    if (wsRef.current) {
      wsRef.current.close();
      wsRef.current = null;
    }
    if (reconnectTimerRef.current) {
      clearTimeout(reconnectTimerRef.current);
      reconnectTimerRef.current = null;
    }
  }, []);

  const sendMessage = useCallback((data: any) => {
    if (wsRef.current?.readyState === WebSocket.OPEN) {
      wsRef.current.send(JSON.stringify(data));
    }
  }, []);

  // 心跳检测
  const startHeartbeat = (ws: WebSocket) => {
    const interval = setInterval(() => {
      if (ws.readyState === WebSocket.OPEN) {
        ws.send('ping');
      } else {
        clearInterval(interval);
      }
    }, config.websocket.heartbeatInterval);
  };

  // 指数退避重连
  const scheduleReconnect = () => {
    if (reconnectAttempts.current >= config.websocket.reconnectAttempts) {
      console.warn('[WebSocket] Max reconnect attempts reached');
      return;
    }

    const delay = config.websocket.reconnectDelay[reconnectAttempts.current] || 16000;
    reconnectTimerRef.current = setTimeout(() => {
      reconnectAttempts.current++;
      connect();
    }, delay);
  };

  useEffect(() => {
    connect();
    return () => disconnect();
  }, [connect, disconnect]);

  return { isConnected, sendMessage, disconnect };
}
```

### 8.3 列表虚拟化

> 📋 增强来源: [[Web聊天界面技术方案]] §5

长会话（>100 条消息）场景下，全量渲染 DOM 节点会导致严重卡顿。使用 `react-window` 的 `VariableSizeList`（非固定高度）实现虚拟滚动：

```tsx
import { useRef, useCallback } from 'react';
import { VariableSizeList } from 'react-window';
import { MessageBubble } from './MessageBubble';

const ESTIMATED_HEIGHT = 120;  // 预估每条消息高度

export function MessageList({ messages, loadMore, hasMore }: MessageListProps) {
  const listRef = useRef<VariableSizeList>(null);
  const sizeMap = useRef<Map<number, number>>(new Map());

  // 动态获取每条消息的真实高度（审批卡片/代码块高度差异大）
  const getItemSize = useCallback((index: number) => {
    return sizeMap.current.get(index) || ESTIMATED_HEIGHT;
  }, []);

  // 消息渲染后更新高度缓存，触发 resetAfterIndex
  const onItemRendered = useCallback((index: number, height: number) => {
    sizeMap.current.set(index, height);
    listRef.current?.resetAfterIndex(index);
  }, []);

  // 新消息到达自动滚动到底部
  const scrollToBottom = useCallback(() => {
    listRef.current?.scrollToItem(messages.length - 1, 'end');
  }, [messages.length]);

  return (
    <VariableSizeList
      ref={listRef}
      height={window.innerHeight - 200}  // 减去 Header + InputArea
      itemCount={messages.length}
      itemSize={getItemSize}
      width="100%"
      onItemsRendered={({ visibleStopIndex }) => {
        // 滚动到顶部时触发加载更早消息
        if (visibleStopIndex === 0 && hasMore) loadMore();
      }}
    >
      {({ index, style }) => (
        <div style={style}>
          <MessageBubble
            message={messages[index]}
            onHeightReady={(h) => onItemRendered(index, h)}
          />
        </div>
      )}
    </VariableSizeList>
  );
}
```

**关键优化点**:
- `VariableSizeList` 替代 `FixedSizeList`：审批卡片（~300px）和普通文本消息（~80px）高度差异大
- `sizeMap` 缓存已渲染消息的真实高度，避免重复测量
- `resetAfterIndex` 仅重置受影响项，O(n) 但 n 为可视区数量
- 仅渲染可视区域 ± overscan（默认 2 条），大幅减少 DOM 节点

- 文件名: 与组件名一致 (例: `MessageBubble.tsx`)
- 函数/变量: `lowerCamelCase` (例: `sendMessage`)
- 常量: `UPPER_SNAKE_CASE` (例: `MAX_MESSAGE_LENGTH`)
- 类型/接口: `UpperCamelCase` (例: `UserInfo`)

### 11.2 代码风格

- 使用 ESLint + Prettier 统一代码风格
- 强制 TypeScript 严格模式
- 禁止 `any` 类型，必须明确类型定义
- 组件必须使用函数式组件 + Hooks

## 十三、总结

本设计方案围绕以下核心原则展开:

1. **前后端分离**: 纯静态资源部署，通过 API 与后端交互
2. **配置化驱动**: 请求域名、超时时间、功能开关全部可配置
3. **模块化架构**: 按业务域拆分模块，清晰的分层结构
4. **类型安全**: TypeScript 全量覆盖，API 接口类型定义
5. **实时通信**: WebSocket + SSE 双通道支持流式对话和实时推送
6. **权限控制**: RBAC 模型，路由级、组件级、按钮级三重控制
7. **性能优化**: 代码分割、虚拟滚动(VariableSizeList)、API 缓存
8. **可扩展性**: 预留国际化、主题定制、PWA 等扩展点
9. **多渠道接入**: IM 适配器架构（企微/钉钉/飞书），统一消息格式
10. **反馈闭环**: 点赞/点踩 → BadCase 工单 → 评测数据集 → 持续优化
11. **审批内嵌**: 审批卡片作为消息类型嵌入聊天流，支持倒计时+断线恢复

通过本方案，可以快速构建一个高质量、可维护、可扩展的企业级 Agent 平台前端应用。
