# P5 前端项目 — 模块级精准生成话术（级别二·增强版）

> 版本: 1.0.0 | 日期: 2026-06-21
> 适用: 基于 `docs/api-documentation/` 的 7 份文档生成 React 前端项目

---

## 使用说明

本文档是一套**完整操作脚本**，按阶段执行即可。每个阶段包含：
- **输入话术**（你发给 AI 的原话，可直接复制粘贴）
- **期望产出**（AI 应该生成什么）
- **验收方法**（你如何确认质量合格）
- **纠错话术**（如果 AI 跑偏了，如何纠正）

---

## 前置约定

### 🔴 铁律（任何阶段不得违反）

| # | 规则 | 说明 |
|:--:|------|------|
| 1 | **文档优先** | 任何设计决策必须以 docs/api-documentation/ 中的原文为准，AI 不得"我觉得" |
| 2 | **逐模块验证** | 每个模块的 tsc 类型检查必须通过，才能进入下一模块 |
| 3 | **类型不改名** | 文档中定义的 interface/type 名称和字段名不得自行简化或重命名 |
| 4 | **API 端点不捏造** | 每个 service 方法必须对应 agent-platform-API.md 中真实存在的端点 |
| 5 | **权限码对齐** | 所有页面/按钮的权限码必须使用 `resource:action` 格式 |

### 文件路径对照

| 文件 | 内容 | 优先级 |
|------|------|:--:|
| `前端架构设计方案.md` | 技术栈、目录结构、配置、路由、状态管理、权限 | P0（总纲） |
| `agent-platform-API.md` | 全部 API 端点、请求/响应 Schema、错误码 | P0（数据源） |
| `前后端权限对应技术方案.md` | RBAC 权限编码、登录流程、Store 设计 | P1（基础设施） |
| `Web聊天界面技术方案.md` | 消息模型、SSE 流式、WebSocket、虚拟滚动 | P1（核心交互） |
| `审批卡片交互技术方案.md` | 审批卡片消息类型、风险等级、倒计时、动画 | P2（聊天内嵌） |
| `用户反馈技术方案.md` | 点赞/点踩、点踩原因、BadCase 工单 | P2（聊天内嵌） |
| `IM接入适配技术方案.md` | 企微/钉钉适配器、统一消息格式 | P3（独立模块） |

### 生成顺序（依赖拓扑）

```
Phase 0: 工程骨架 ── 一次性
    │
Phase 1: 认证 + 权限基础设施 ── 所有模块的前置依赖
    │
Phase 2: 用户/角色/权限/租户管理 ── 后台管理基础
    │
Phase 3: 会话 + 消息 + Web 聊天界面 ── 核心交互
    │
Phase 4: 审批卡片 + 用户反馈 ── 聊天内嵌组件
    │
Phase 5: 知识库 + 文档 + 检索 + 工具平台 + 任务 + 提示词 ── 业务模块
    │
Phase 6: 安全围栏 + 意图 + 效果评估 + 优化工单 + 可观测性 ── 运营模块
    │
Phase 7: IM 适配 ── 独立渠道层
```

> ⚠️ **严禁跳阶段**。Phase 3 的聊天界面依赖 Phase 1 的认证和 Phase 2 的用户体系，如果跳过 Phase 1-2 直接做 Phase 3，生成的消息组件会因为没有 token 注入、没有 useAuthStore 而变成废代码。
