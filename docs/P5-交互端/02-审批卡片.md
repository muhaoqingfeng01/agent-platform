# P5-02 审批卡片

## 概述
在人机协同场景中，AI Agent 执行敏感操作前需要人工审批。审批卡片作为自定义消息类型嵌入聊天流中。

## 消息格式
```json
{
  "type": "approval",
  "approvalId": "uuid",
  "title": "删除用户账号",
  "detail": "即将删除用户 [张三] 及其关联数据，不可恢复",
  "riskLevel": "high",
  "options": ["同意", "拒绝"],
  "timeout": 300,
  "createdAt": "2026-06-13T12:00:00+08:00"
}
```

## 前端渲染
- 卡片组件 `ApprovalCard` 内嵌在消息列表中
- 显示业务详情 + "同意"/"拒绝"按钮
- 超时倒计时展示（超时后按钮禁用）
- 高风险操作红色边框警示

## 交互流程
1. 用户点击"同意"/"拒绝"
2. 前端通过 WebSocket 发送 `approval_response` 消息
3. 携带 `approvalId` + `action`（approve/reject）
4. 后端处理后通过 WebSocket 推送处理结果
