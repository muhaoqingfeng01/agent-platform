# 用户管理与登录认证

## 所属阶段
**P0 基础底座 → T8 多租户数据与存储**

## 使用技术
- BCrypt 密码加密
- Sa-Token 登录（StpUtil.login）
- Sa-Token `UserDetailsService`

## 涉及数据库表
- `t_user`

## API 端点

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | `/api/v1/auth/register` | 用户注册 | 无 |
| POST | `/api/v1/auth/login` | 登录，返回 Token | 无 |
| POST | `/api/v1/auth/refresh` | 刷新 AccessToken | RefreshToken |
| POST | `/api/v1/auth/logout` | 登出（Token 加黑名单） | Bearer |
| GET | `/api/v1/auth/me` | 获取当前用户信息 | Bearer |
| GET | `/api/v1/users` | 用户列表 | ADMIN |
| GET | `/api/v1/users/{id}` | 用户详情 | ADMIN/本人 |
| PUT | `/api/v1/users/{id}` | 更新用户信息 | ADMIN/本人 |
| PATCH | `/api/v1/users/{id}/status` | 启停用户 | ADMIN |
| PATCH | `/api/v1/users/{id}/password` | 修改密码 | 本人 |

## 实现方案

### 1. 登录流程

```
1. 用户 POST /auth/login {username, password}
2. AuthService.loadUserByUsername() → UserDetails
3. BCrypt.matches(password, hash) → 校验
4. StpUtil 生成 AccessToken + RefreshToken
5. RefreshToken 存入 Redis(key=refresh:{userId}, TTL=7d)
6. 返回 {accessToken, refreshToken, expiresIn}
```

### 2. 注册流程

```
1. POST /auth/register {tenantId, username, password, email}
2. 校验租户存在且 ACTIVE
3. 校验用户名同租户下唯一
4. BCrypt 加密密码（strength=12）
5. 生成 user_id，写入 t_user 表
6. 分配默认角色（VIEWER）
7. 返回注册成功
```

### 3. 认证服务

```java
@Service
public class AuthService {

    public LoginResponse login(LoginRequest request) {
        // 1. 校验用户状态
        User user = userRepository.findByTenantAndUsername(request.getTenantId(), request.getUsername());
        if (t_user == null || user.isDeleted()) throw new BadCredentialsException("用户不存在");
        if (user.isDisabled()) throw new AccountStatusException("账户已停用");

        // 2. 校验密码
        if (!passwordEncoder.matches(request.getPassword(), user.getPasswordHash())) {
            throw new BadCredentialsException("密码错误");
        }

        // 3. 生成 Token
                // Sa-Token 登录，自动生成 Token
        StpUtil.login(user.getUserId());
        redisTemplate.opsForValue().set("refresh:" + user.getUserId(), refreshToken, 7, TimeUnit.DAYS);

        return new LoginResponse(accessToken, refreshToken, 3600);
    }
}
```

### 4. 密码策略

| 规则 | 说明 |
|------|------|
| 最小长度 | 8 位 |
| 复杂度 | 至少包含大写、小写、数字、特殊字符中的 3 种 |
| 加密算法 | BCrypt(strength=12) |
| 修改密码 | 需验证旧密码，修改后所有 Token 失效 |
