# 会话管理

## 所属阶段
**P1 核心能力 → T3 意图识别与对话管理**

## 使用技术
- Spring Web MVC
- MyBatis + XML Mapper（单表查询，禁止 JOIN）
- Redis（活跃会话缓存 + 分布式锁）
- Sa-Token（认证 + 租户上下文）

## 涉及数据库表
- `t_conversation` — 会话主表
- `t_message` — 关联消息（参见 02-消息收发）

## 设计模式应用

| 设计模式 | 应用场景 | 核心价值 |
|----------|----------|----------|
| **Builder** | 会话实体构建、响应体构建 | 链式构造，避免冗长构造函数 |
| **State** | 会话状态机（ACTIVE/CLOSED/ARCHIVED） | 状态转移显式化，禁止非法转移 |
| **Repository** | 会话数据访问抽象 | 领域层不感知持久化细节 |
| **Strategy** | 缓存策略（Redis 优先 → MySQL 回退） | 策略可替换，读写分离 |
| **Template Method** | 基础 CRUD 骨架（校验→处理→响应） | 统一流程，子类只填差异 |
| **Factory Method** | IdGenerator 分布式 ID 生成 | 隔离 ID 生成策略 |
| **Observer** | 会话变更事件推送（WebSocket 通知） | 解耦会话操作与通知 |
| **Decorator** | 缓存层装饰 Repository | 透明加缓存，不侵入原逻辑 |

### 2. 领域实体（Builder + State 模式）

```java
/**
 * 会话聚合根 — 封装会话生命周期与业务不变量.
 * <p>
 * 使用 Builder 模式构建，State 模式管理状态流转。
 *
 * @author Agent Platform Team
 * @since 1.0.0
 */
@Getter
@Builder
public class Conversation {

    /** 会话唯一标识（conv_ 前缀 + 雪花算法） */
    private String conversationId;

    /** 所属租户 */
    private String tenantId;

    /** 关联 Agent 配置 */
    private String agentId;

    /** 发起用户 */
    private String userId;

    /** 会话标题 */
    private String title;

    /** 会话状态 */
    private ConversationStatus status;

    /** 消息计数 */
    private int messageCount;

    /** 累计 Token 消耗 */
    private long totalTokens;

    /** 扩展元数据（JSON） */
    private Map<String, Object> metadata;

    /** 创建/更新时间 */
    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;

    // ========== State 模式：状态流转 ==========

    /**
     * 关闭会话 — 仅 ACTIVE 状态可关闭.
     */
    public void close() {
        if (this.status != ConversationStatus.ACTIVE) {
            throw new IllegalStateException(
                "仅 ACTIVE 状态可关闭，当前: " + this.status);
        }
        this.status = ConversationStatus.CLOSED;
    }

    /**
     * 重开会话 — 仅 CLOSED 状态可重开.
     */
    public void reopen() {
        if (this.status != ConversationStatus.CLOSED) {
            throw new IllegalStateException(
                "仅 CLOSED 状态可重开，当前: " + this.status);
        }
        this.status = ConversationStatus.ACTIVE;
    }

    /**
     * 归档会话 — ACTIVE 或 CLOSED 均可归档，归档后不可逆.
     */
    public void archive() {
        if (this.status == ConversationStatus.ARCHIVED) {
            throw new IllegalStateException("已归档的会话不可再次归档");
        }
        this.status = ConversationStatus.ARCHIVED;
    }

    /**
     * 增量更新消息计数和 Token.
     */
    public void incrementMessage(int tokenCount) {
        this.messageCount++;
        this.totalTokens += tokenCount;
    }

    /**
     * 更新标题（仅当非空且不同于当前标题）.
     */
    public void updateTitle(String newTitle) {
        if (newTitle != null && !newTitle.isBlank() && !newTitle.equals(this.title)) {
            this.title = newTitle;
        }
    }
}
```

```
                    │    │ reopen()       │           │
                    │    ▼                │           │
```

```java
/**
 * 会话状态值对象 — State 模式.
 * <p>
 * 每个状态枚举自带合法的状态转移方法，非法转移在编译期即可发现.
 */
@Getter
@AllArgsConstructor
public enum ConversationStatus {

    /** 活跃 — 可收发消息 */
    ACTIVE("活跃") {
        @Override
        public Set<ConversationStatus> allowedTransitions() {
            return EnumSet.of(CLOSED, ARCHIVED);
        }
    },

    /** 已关闭 — 不可收发，可重开或归档 */
    CLOSED("已关闭") {
        @Override
        public Set<ConversationStatus> allowedTransitions() {
            return EnumSet.of(ACTIVE, ARCHIVED);
        }
    },

    /** 已归档 — 只读，不可逆 */
    ARCHIVED("已归档") {
        @Override
        public Set<ConversationStatus> allowedTransitions() {
            return EnumSet.noneOf(ConversationStatus.class);
        }
    };

    private final String label;

    /** 当前状态允许转移到的目标状态集合 */
    public abstract Set<ConversationStatus> allowedTransitions();

    /**
     * 校验状态转移是否合法.
     */
    public boolean canTransitionTo(ConversationStatus target) {
        return allowedTransitions().contains(target);
    }
}
```

### 5. Application Service（Template Method + Facade 模式）

```java
/**
 * 会话应用服务 — Facade 模式对外暴露，Template Method 定义骨架.
 * <p>
 * 统一的"校验 → 处理 → 持久化 → 通知 → 返回"五步骨架.
 *
 * @author Agent Platform Team
 * @since 1.0.0
 */
@Service
@Slf4j
@RequiredArgsConstructor
public class ConversationApplicationService {

    private final ConversationRepository conversationRepository;
    private final ConversationDomainService domainService;
    private final CachedConversationRepository cachedRepository;  // Decorator
    private final ConversationEventPublisher eventPublisher;      // Observer

    /**
     * 创建会话 — Template Method 骨架.
     */
    @Transactional
    public ConversationResponse createConversation(CreateConversationRequest request) {
        // 1. 校验
        validateCreateRequest(request);

        // 2. 构建领域实体（Builder 模式）
        Conversation conversation = Conversation.builder()
                .conversationId(IdGenerator.generate("conv"))
                .tenantId(TenantContext.getCurrentTenantId())
                .agentId(request.getAgentId())
                .userId(TenantContext.getCurrentUserId())
                .title(request.getTitle() != null ? request.getTitle() : "新对话")
                .status(ConversationStatus.ACTIVE)
                .messageCount(0)
                .totalTokens(0L)
                .metadata(request.getMetadata() != null ? request.getMetadata() : new HashMap<>())
                .createdAt(LocalDateTime.now())
                .updatedAt(LocalDateTime.now())
                .build();

        // 3. 领域校验
        domainService.validateNewConversation(conversation);

        // 4. 持久化
        conversationRepository.save(conversation);

        // 5. 缓存预热（Decorator 模式）
        cachedRepository.cacheActiveConversation(conversation);

        // 6. 事件通知（Observer 模式）
        eventPublisher.publishConversationCreated(conversation);

        log.info("[Conversation] 创建会话: convId={}, userId={}, agentId={}",
                conversation.getConversationId(), conversation.getUserId(), conversation.getAgentId());

        // 7. 组装响应
        return ConversationAssembler.toResponse(conversation);
    }

    /**
     * 查询会话列表 — Strategy 模式：缓存优先 → DB 回退.
     */
    public PageResponse<ConversationResponse> listConversations(int page, int size) {
        String tenantId = TenantContext.getCurrentTenantId();
        String userId = TenantContext.getCurrentUserId();

        // Strategy 1: 从 Redis 读取活跃会话（最近 20 个）
        List<Conversation> cached = cachedRepository.findActiveConversations(tenantId, userId, 20);
        if (!cached.isEmpty() && page == 1 && cached.size() >= size) {
            return PageResponse.of(cached.stream().map(ConversationAssembler::toResponse).toList(),
                    cached.size(), page, size);
        }

        // Strategy 2: MySQL 分页查询
        List<Conversation> conversations = conversationRepository.findByUserId(tenantId, userId, page, size);
        long total = conversationRepository.countByUserId(tenantId, userId);

        return PageResponse.of(
                conversations.stream().map(ConversationAssembler::toResponse).toList(),
                total, page, size);
    }

    /**
     * 状态流转 — State 模式驱动.
     */
    @Transactional
    public void transitionStatus(String conversationId, ConversationStatus targetStatus) {
        Conversation conversation = conversationRepository.findByConversationId(conversationId)
                .orElseThrow(() -> new ResourceNotFoundException("会话", conversationId));

        // State 模式：校验状态转移合法性
        switch (targetStatus) {
            case CLOSED  -> conversation.close();
            case ACTIVE  -> conversation.reopen();
            case ARCHIVED -> conversation.archive();
            default -> throw new BusinessException(400, "不支持的目标状态: " + targetStatus);
        }

        conversationRepository.updateStatus(conversationId, targetStatus);
        cachedRepository.evictActiveConversation(conversation.getTenantId(), conversation.getUserId());

        log.info("[Conversation] 状态变更: convId={}, {} → {}",
                conversationId, conversation.getStatus(), targetStatus);
    }

    private void validateCreateRequest(CreateConversationRequest request) {
        Objects.requireNonNull(request.getAgentId(), "agentId 不能为空");
    }
}
```

### 6. 缓存装饰器（Decorator 模式）

```java
/**
 * 会话缓存装饰器 — 透明地为 Repository 添加 Redis 缓存层.
 * <p>
 * 遵循 Decorator 模式：实现同一接口，组合原始 Repository，增强其行为.
 *
 * @author Agent Platform Team
 * @since 1.0.0
 */
@Component
@Slf4j
@RequiredArgsConstructor
public class CachedConversationRepository {

    private static final String CACHE_KEY_PREFIX = "conv:active:";
    private static final int MAX_CACHED_CONVERSATIONS = 20;
    private static final Duration CACHE_TTL = Duration.ofMinutes(10);

    private final RedisTemplate<String, String> redisTemplate;
    private final ObjectMapper objectMapper;

    /**
     * 缓存活跃会话列表.
     */
    public void cacheActiveConversation(Conversation conversation) {
        String key = buildCacheKey(conversation.getTenantId(), conversation.getUserId());
        try {
            String json = objectMapper.writeValueAsString(ConversationAssembler.toCacheEntry(conversation));
            redisTemplate.opsForList().leftPush(key, json);
            redisTemplate.opsForList().trim(key, 0, MAX_CACHED_CONVERSATIONS - 1);
            redisTemplate.expire(key, CACHE_TTL);
        } catch (JsonProcessingException e) {
            log.warn("[Conversation] 缓存序列化失败: convId={}", conversation.getConversationId(), e);
        }
    }

    /**
     * 从缓存读取活跃会话.
     */
    public List<Conversation> findActiveConversations(String tenantId, String userId, int count) {
        String key = buildCacheKey(tenantId, userId);
        List<String> entries = redisTemplate.opsForList().range(key, 0, count - 1);
        if (entries == null || entries.isEmpty()) {
            return List.of();
        }
        return entries.stream()
                .map(json -> {
                    try {
                        return objectMapper.readValue(json, ConversationCacheEntry.class).toConversation();
                    } catch (JsonProcessingException e) {
                        log.warn("[Conversation] 缓存反序列化失败", e);
                        return null;
                    }
                })
                .filter(Objects::nonNull)
                .toList();
    }

    /**
     * 失效缓存.
     */
    public void evictActiveConversation(String tenantId, String userId) {
        redisTemplate.delete(buildCacheKey(tenantId, userId));
    }

    private String buildCacheKey(String tenantId, String userId) {
        return CACHE_KEY_PREFIX + tenantId + ":" + userId;
    }
}
```

### 7. Controller（REST 入口）

```java
/**
 * 会话管理 Controller.
 *
 * @author Agent Platform Team
 * @since 1.0.0

@RestController
@RequestMapping("/api/v1/conversations")
@Tag(name = "对话管理", description = "会话 CRUD 与状态管理")
@Slf4j
@RequiredArgsConstructor
public class ConversationController {

    private final ConversationApplicationService applicationService;

    @PostMapping
    @SaCheckPermission("conversation:create")
    @Operation(summary = "创建新会话")
    public Result<ConversationResponse> create(@Valid @RequestBody CreateConversationRequest request) {
        log.info("[Conversation] 创建会话: agentId={}", request.getAgentId());
        ConversationResponse response = applicationService.createConversation(request);
        return Result.ok(response);
    }

    @GetMapping
    @SaCheckPermission("conversation:read")
    @Operation(summary = "我的会话列表")
    public Result<PageResponse<ConversationResponse>> list(
            @RequestParam(defaultValue = "1") int page,
            @RequestParam(defaultValue = "20") int size) {
        return Result.ok(applicationService.listConversations(page, size));
    }

    @GetMapping("/{id}")
    @SaCheckPermission("conversation:read")
    @Operation(summary = "会话详情")
    public Result<ConversationResponse> getById(@PathVariable String id) {
        Conversation conversation = applicationService.getConversation(id);
        return Result.ok(ConversationAssembler.toResponse(conversation));
    }

    @PatchMapping("/{id}/title")
    @SaCheckPermission("conversation:update")
    @Operation(summary = "更新标题")
    public Result<Void> updateTitle(@PathVariable String id,
                                     @Valid @RequestBody UpdateTitleRequest request) {
        applicationService.updateTitle(id, request.getTitle());
        return Result.ok();
    }

    @PatchMapping("/{id}/status")
    @SaCheckPermission("conversation:update")
    @Operation(summary = "状态流转（close/reopen/archive）")
    public Result<Void> transitionStatus(@PathVariable String id,
                                          @Valid @RequestBody TransitionStatusRequest request) {
        applicationService.transitionStatus(id, request.getTargetStatus());
        return Result.ok();
    }

    @DeleteMapping("/{id}")
    @SaCheckPermission("conversation:delete")
    @Operation(summary = "逻辑删除")
    public Result<Void> delete(@PathVariable String id) {
        applicationService.softDelete(id);
        return Result.ok();
    }
}
```

### 9. 事件通知（Observer 模式）

```java
/**
 * 会话事件发布器 — Observer 模式中的 Subject.
 * <p>
 * 会话操作完成后，异步通知所有注册的监听器（WebSocket 推送、审计日志等）.

@Component
@RequiredArgsConstructor
public class ConversationEventPublisher {

    private final ApplicationEventPublisher springEventPublisher;

    /**
     * 发布"会话创建"事件.

    public void publishConversationCreated(Conversation conversation) {
        springEventPublisher.publishEvent(new ConversationCreatedEvent(this, conversation));
    }

    /**
     * 发布"会话状态变更"事件.

    public void publishStatusChanged(Conversation conversation, ConversationStatus oldStatus) {
        springEventPublisher.publishEvent(
                new ConversationStatusChangedEvent(this, conversation, oldStatus));

/**
 * 会话事件监听器 — Observer 模式中的 Observer.
 * <p>
 * 异步处理，不阻塞主流程.
 */
@Component
@Slf4j
public class ConversationEventListener {

    private final ConversationWebSocketHandler webSocketHandler;

    @EventListener
    @Async
    public void onConversationCreated(ConversationCreatedEvent event) {
        Conversation conv = event.getConversation();
        webSocketHandler.pushMessage(conv.getUserId(), WebSocketMessage.builder()
                .type("conversation_created")
                .payload(ConversationAssembler.toResponse(conv))
                .timestamp(System.currentTimeMillis())
                .build());
        log.debug("[Conversation] WebSocket 推送创建事件: convId={}", conv.getConversationId());
    }

    @EventListener
    @Async
    public void onStatusChanged(ConversationStatusChangedEvent event) {
        Conversation conv = event.getConversation();
        webSocketHandler.pushMessage(conv.getUserId(), WebSocketMessage.builder()
                .type("conversation_status_changed")
                .payload(Map.of(
                        "conversationId", conv.getConversationId(),
                        "oldStatus", event.getOldStatus(),
                        "newStatus", conv.getStatus()
                ))
                .timestamp(System.currentTimeMillis())
                .build());
    }
}
```

## 关键设计决策

1. **State 模式管理状态而非 if-else** — 每个状态枚举自带合法转移集合，开发者无法写出非法转移逻辑
2. **Decorator 模式做缓存** — CachedConversationRepository 透明增强原始 Repository，不影响单元测试
3. **Observer 模式解耦通知** — 会话操作完成后通过 Spring Event 异步通知 WebSocket/审计等，不阻塞主流程
4. **Builder 模式构建实体** — 8 个必填字段，避免冗长构造函数，确保不可变对象构建正确
5. **Template Method 统一 CRUD 骨架** — 每个 ApplicationService 方法遵循 `校验→处理→持久化→通知→返回` 五步
6. **Redis List 做最近会话缓存** — TTL 10 分钟，LEFT PUSH + TRIM 保持最新 20 条，避免缓存膨胀
