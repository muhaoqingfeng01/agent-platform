# 会话管理

## 所属阶段
**P1 核心能力 → T3 意图识别与对话管理**

## 使用技术
- Spring Web MVC
- MyBatis-Plus
- Redis（活跃会话缓存）

## 涉及数据库表
- `t_conversation`

## API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/conversations` | 创建新会话 |
| GET | `/api/v1/conversations` | 我的会话列表（分页） |
| GET | `/api/v1/conversations/{id}` | 会话详情（含元数据） |
| PATCH | `/api/v1/conversations/{id}/title` | 更新标题 |
| PATCH | `/api/v1/conversations/{id}/close` | 关闭会话 |
| PATCH | `/api/v1/conversations/{id}/archive` | 归档会话 |
| DELETE | `/api/v1/conversations/{id}` | 逻辑删除 |

## 实现方案

### 1. 会话创建

```java
@Service
public class ConversationService {

    public ConversationResponse createConversation(CreateConversationRequest request) {
        String conversationId = IdGenerator.generate("conv");
        Conversation conv = Conversation.builder()
                .conversationId(conversationId)
                .tenantId(TenantContext.getCurrentTenantId())
                .agentId(request.getAgentId())
                .userId(TenantContext.getCurrentUserId())
                .title(request.getTitle() != null ? request.getTitle() : "新对话")
                .status(ConversationStatus.ACTIVE)
                .messageCount(0)
                .totalTokens(0L)
                .build();
        conversationRepository.save(conv);
        return ConversationResponse.from(conv);
    }
}
```

### 2. 会话状态机

```
ACTIVE ──关闭──→ CLOSED ──归档──→ ARCHIVED
  │                                  │
  └────────── 直接归档 ──────────────┘
```

| 状态 | 说明 |
|------|------|
| ACTIVE | 可收发消息 |
| CLOSED | 不再接受新消息，可重新打开 |
| ARCHIVED | 只读，不可恢复 |

### 3. 会话列表缓存策略

```
用户最近 20 个活跃会话 → Redis List (TTL=10min)
历史会话 → MySQL 分页查询
```
