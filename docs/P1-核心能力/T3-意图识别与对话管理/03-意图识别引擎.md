# 意图识别引擎

## 所属阶段
**P1 核心能力 → T3 意图识别与对话管理**

## 使用技术
- Spring AI `ChatClient`（LLM 意图分类）
- 正则/关键词匹配引擎（规则层）
- Redis（意图缓存 + 相似问法向量缓存）
- MyBatis + XML Mapper（意图定义持久化）

## 涉及数据库表
- `t_intent` — 意图定义表

## 设计模式应用

| 设计模式 | 应用场景 | 核心价值 |
|----------|----------|----------|
| **Chain of Responsibility** | 三层识别链路（规则→缓存→LLM） | 每层独立，命中即短路，新增识别策略不改链路 |
| **Strategy** | 规则匹配策略（正则/关键词/混合） | 不同匹配算法可插拔替换 |
| **Template Method** | 意图 CRUD 骨架（校验→处理→持久化→缓存失效） | 统一流程，子类填差异 |
| **Builder** | IntentResult 构建 | 链式构造识别结果 |
| **Command** | 意图测试 API（输入文本 → 匹配结果） | 封装请求为对象，支持异步/队列 |
| **Factory** | IntentRecognizer 工厂（根据 category 选择策略） | 隔离创建逻辑 |
| **Decorator** | 缓存装饰器包装 LLM 识别器 | 透明添加缓存，可插拔 |
| **Observer** | 意图变更通知（启停/编辑 → 缓存失效 → 通知各识别层） | 解耦意图管理面与控制面 |

### 1. 混合识别架构（Chain of Responsibility）

```
用户输入: "帮我查一下最近的订单"
    │
    ▼
┌────────────────────────────────────────────────────┐
│  IntentRecognitionChain (Chain of Responsibility) │
│                                                    │
│  ┌──────────────────┐                              │
│  │ Layer 1: 规则匹配  │ ← 正则 + 关键词（~1ms）       │
│  │ RuleRecognizer    │   命中且 confidence ≥ 0.9    │
│  └────────┬─────────┘    → 短路返回                  │
│           │ 未命中                                    │
│           ▼                                         │
│  ┌──────────────────┐                              │
│  │ Layer 2: Redis缓存 │ ← 相似问法缓存（~5ms）         │
│  │ CacheRecognizer   │   命中 → 短路返回              │
│  └────────┬─────────┘                              │
│           │ 未命中                                    │
│           ▼                                         │
│  ┌──────────────────┐                              │
│  │ Layer 3: LLM 分类  │ ← 兜底策略（~500ms）           │
│  │ LLMRecognizer     │   调用 LLM + Few-shot        │
│  └────────┬─────────┘                              │
│           │                                         │
│           ▼                                         │
│  ┌──────────────────┐                              │
│  │ 结果后处理 + 缓存  │ ← 写入 Redis（TTL=30min）      │
│  └──────────────────┘                              │
└────────────────────────────────────────────────────┘
    │
    ▼
  IntentResult { intentCode, intentName, confidence, reasoning }
```

### 2. 责任链核心实现（Chain of Responsibility 模式）

```java
/**
 * 意图识别链 — Chain of Responsibility 核心.
 * <p>
 * 三层识别策略串联：规则优先（高速）→ 缓存中间（中速）→ LLM 兜底（低速）.
 * 每层命中即短路，未命中则沿链继续传递.
 *
 * @author Agent Platform Team
 * @since 1.0.0
 */
@Service
@Slf4j
@RequiredArgsConstructor
public class IntentRecognitionChain {

    private final RuleRecognizer ruleRecognizer;
    private final CacheRecognizer cacheRecognizer;
    private final LLMRecognizer llmRecognizer;

    /**
     * 执行意图识别 — 责任链模式.
     * <p>
     * 链中每个节点返回 Optional.empty() 表示"未命中，继续下一层"，
     * 返回 Optional.of(result) 表示"命中，短路返回".
     *
     * @param userInput 用户输入文本
     * @return 识别结果（保证非 null，未匹配到任何意图时返回 UNKNOWN）
     */
    public IntentResult recognize(String userInput) {
        long startTime = System.currentTimeMillis();
        String tenantId = TenantContext.getCurrentTenantId();

        try {
            // Layer 1: 规则匹配（~1ms）
            Optional<IntentResult> ruleResult = ruleRecognizer.recognize(tenantId, userInput);
            if (ruleResult.isPresent() && ruleResult.get().getConfidence() >= 0.9) {
                log.debug("[Intent] Layer 1 命中: intent={}, confidence={}, cost={}ms",
                        ruleResult.get().getIntentCode(), ruleResult.get().getConfidence(),
                        System.currentTimeMillis() - startTime);
                return ruleResult.get();
            }

            // Layer 2: Redis 缓存（~5ms）
            Optional<IntentResult> cachedResult = cacheRecognizer.recognize(tenantId, userInput);
            if (cachedResult.isPresent()) {
                log.debug("[Intent] Layer 2 命中: intent={}, cost={}ms",
                        cachedResult.get().getIntentCode(), System.currentTimeMillis() - startTime);
                return cachedResult.get();
            }

            // Layer 3: LLM 兜底（~500ms）
            IntentResult llmResult = llmRecognizer.recognize(tenantId, userInput);

            // 后处理：结果写入缓存（Decorator 模式）
            cacheRecognizer.cacheResult(tenantId, userInput, llmResult);

            log.info("[Intent] Layer 3 兜底: intent={}, confidence={}, cost={}ms",
                    llmResult.getIntentCode(), llmResult.getConfidence(),
                    System.currentTimeMillis() - startTime);

            return llmResult;

        } catch (Exception e) {
            log.error("[Intent] 识别异常: input={}", userInput, e);
            // 降级策略：返回 UNKNOWN（不阻塞对话流程）
            return IntentResult.unknown(e.getMessage());
        }
    }
}
```

### 3. Strategy 模式 — 规则匹配器

```java
/**
 * 规则匹配器 — Strategy 模式封装不同匹配算法.
 * <p>
 * 支持三种匹配策略：正则匹配、关键词匹配、混合加权匹配.
 *
 * @author Agent Platform Team
 * @since 1.0.0
 */
@Component
@Slf4j
public class RuleRecognizer {

    private final IntentRepository intentRepository;

    public RuleRecognizer(IntentRepository intentRepository) {
        this.intentRepository = intentRepository;
    }

    /**
     * 规则识别入口 — Strategy 模式选择匹配策略.
     *
     * @param tenantId  租户 ID
     * @param userInput 用户输入
     * @return 识别结果，未命中返回 empty
     */
    public Optional<IntentResult> recognize(String tenantId, String userInput) {
        List<Intent> intents = intentRepository.findActiveByTenant(tenantId);
        if (intents.isEmpty()) {
            return Optional.empty();
        }

        String normalizedInput = normalize(userInput);

        for (Intent intent : intents) {
            List<String> patterns = intent.getPatterns();
            if (patterns == null || patterns.isEmpty()) {
                continue;
            }

            // Strategy: 先尝试正则精确匹配
            for (String pattern : patterns) {
                if (isRegexPattern(pattern)) {
                    if (Pattern.matches(extractRegex(pattern), normalizedInput)) {
                        return Optional.of(IntentResult.matched(intent, 1.0,
                                "正则匹配: " + pattern));
                    }
                }
            }

            // Strategy: 关键词加权匹配
            double keywordScore = calculateKeywordScore(patterns, normalizedInput);
            if (keywordScore >= 0.7) {
                return Optional.of(IntentResult.matched(intent, keywordScore,
                        "关键词匹配: score=" + String.format("%.2f", keywordScore)));
            }
        }
        return Optional.empty();
    }

    /**
     * 判断是否为正则表达式模式（以 / 包裹）.
     */
    private boolean isRegexPattern(String pattern) {
        return pattern.startsWith("/") && pattern.endsWith("/");
    }

    /**
     * 提取正则表达式内容（去掉首尾的 /）.
     */
    private String extractRegex(String pattern) {
        return pattern.substring(1, pattern.length() - 1);
    }

    /**
     * 关键词加权匹配 — 每个匹配关键词加分，根据密度计算最终置信度.
     *
     * <pre>
     *   置信度 = 匹配关键词数 / 总关键词数 × 密度因子
     *   密度因子 = 1 + (匹配字符数 / 输入字符数) × 0.2
     *   上限 0.9（关键词匹配最高置信度）
     * </pre>
     */
    private double calculateKeywordScore(List<String> patterns, String input) {
        int totalKeywords = 0;
        int matchedKeywords = 0;
        int matchedChars = 0;

        for (String pattern : patterns) {
            if (!isRegexPattern(pattern)) {
                totalKeywords++;
                if (input.contains(pattern)) {
                    matchedKeywords++;
                    matchedChars += pattern.length();
                }
            }
        }

        if (totalKeywords == 0 || matchedKeywords == 0) {
            return 0.0;
        }

        double baseScore = (double) matchedKeywords / totalKeywords;
        double densityFactor = 1.0 + ((double) matchedChars / input.length()) * 0.2;
        return Math.min(baseScore * densityFactor, 0.9);
    }

    /**
     * 输入标准化：去标点、去多余空格、小写化.
     */
    private String normalize(String input) {
        return input.replaceAll("[，。！？、；：\"'（）【】《》\\s]+", "")
                .trim()
                .toLowerCase();
    }
}
```

### 4. 缓存识别器（Decorator 模式透明增强）

```java
/**
 * 缓存识别器 — Decorator 模式为 LLM 识别器添加缓存层.
 * <p>
 * 使用 Redis 缓存最近识别结果，相似输入直接返回缓存结果，减少 LLM 调用.
 *
 * @author Agent Platform Team
 * @since 1.0.0
 */
@Component
@Slf4j
@RequiredArgsConstructor
public class CacheRecognizer {

    private static final String CACHE_NAMESPACE = "intent:cache";
    private static final Duration CACHE_TTL = Duration.ofMinutes(30);

    private final RedisTemplate<String, String> redisTemplate;
    private final ObjectMapper objectMapper;

    /**
     * 尝试从缓存获取识别结果.
     */
    public Optional<IntentResult> recognize(String tenantId, String userInput) {
        String cacheKey = buildCacheKey(tenantId, userInput);
        String cached = redisTemplate.opsForValue().get(cacheKey);
        if (cached != null) {
            try {
                IntentResult result = objectMapper.readValue(cached, IntentResult.class);
                log.debug("[Intent] 缓存命中: key={}", cacheKey);
                return Optional.of(result);
            } catch (JsonProcessingException e) {
                log.warn("[Intent] 缓存反序列化失败", e);
            }
        }
        return Optional.empty();
    }

    /**
     * 缓存识别结果.
     */
    public void cacheResult(String tenantId, String userInput, IntentResult result) {
        try {
            String cacheKey = buildCacheKey(tenantId, userInput);
            String json = objectMapper.writeValueAsString(result);
            redisTemplate.opsForValue().set(cacheKey, json, CACHE_TTL);
            log.debug("[Intent] 结果缓存: key={}, intent={}", cacheKey, result.getIntentCode());
        } catch (JsonProcessingException e) {
            log.warn("[Intent] 缓存序列化失败", e);
        }
    }

    /**
     * 失效缓存（意图定义变更时调用 — Observer 模式）.
     */
    public void evictCache(String tenantId) {
        String pattern = CACHE_NAMESPACE + ":" + tenantId + ":*";
        Set<String> keys = redisTemplate.keys(pattern);
        if (keys != null && !keys.isEmpty()) {
            redisTemplate.delete(keys);
            log.info("[Intent] 缓存失效: tenantId={}, count={}", tenantId, keys.size());
        }
    }

    private String buildCacheKey(String tenantId, String userInput) {
        // 使用输入文本的 hash 作为 key 后缀，避免 key 过长
        String inputHash = Integer.toHexString(userInput.trim().toLowerCase().hashCode());
        return CACHE_NAMESPACE + ":" + tenantId + ":" + inputHash;
    }
}
```

### 5. LLM 兜底识别器（Template Method 模式）

```java
/**
 * LLM 意图分类器 — Template Method 定义分类骨架.
 * <p>
 * 模板方法 define() 定义步骤：加载意图 → 构建 Prompt → 调用 LLM → 解析结果.
 * 子类可覆盖 buildPrompt() 实现不同的 Prompt 策略.
 *
 * @author Agent Platform Team
 * @since 1.0.0
 */
@Component
@Slf4j
@RequiredArgsConstructor
public class LLMRecognizer {

    private final IntentRepository intentRepository;
    private final ChatClient chatClient;
    private final ObjectMapper objectMapper;

    /**
     * Template Method: LLM 分类骨架.
     */
    public IntentResult recognize(String tenantId, String userInput) {
        // Step 1: 加载所有活跃意图
        List<Intent> intents = intentRepository.findActiveByTenant(tenantId);
        if (intents.isEmpty()) {
            return IntentResult.unknown("当前租户未配置任何意图");
        }

        // Step 2: 构建 Few-shot 分类 Prompt（子类可覆盖）
        String prompt = buildClassificationPrompt(intents, userInput);

        // Step 3: 调用 LLM
        try {
            String response = chatClient.prompt()
                    .user(prompt)
                    .options(OpenAiChatOptions.builder()
                            .withTemperature(0.1)  // 低温度增强确定性
                            .withMaxTokens(256)     // 限制输出长度
                            .build())
                    .call()
                    .content();

            // Step 4: 解析 LLM 输出
            return parseIntentFromLLM(response, intents);

        } catch (Exception e) {
            log.error("[Intent] LLM 分类失败", e);
            // 降级：返回 UNKNOWN，不阻塞对话
            return IntentResult.unknown("LLM 分类失败: " + e.getMessage());
        }
    }

    /**
     * 构建 Few-shot 分类提示词 — 模板方法，可被子类覆盖.
     *
     * <pre>
     *   提示词结构：
     *   1. 角色定义（你是一个意图分类器）
     *   2. 意图列表（含示例问法 ≈ Few-shot）
     *   3. 用户输入
     *   4. 输出格式要求（JSON Schema）
     * </pre>
     */
    protected String buildClassificationPrompt(List<Intent> intents, String userInput) {
        StringBuilder intentsBlock = new StringBuilder();
        for (Intent intent : intents) {
            intentsBlock.append("""
                - **编码**: %s
                  **名称**: %s
                  **分类**: %s
                  **描述**: %s
                  **示例问法**: %s
                """.formatted(
                    intent.getIntentCode(),
                    intent.getIntentName(),
                    intent.getCategory(),
                    intent.getLlmPrompt() != null ? intent.getLlmPrompt() : "无",
                    intent.getExamples() != null ? String.join(", ", intent.getExamples()) : "无"
                ));
        }

        return """
            你是一个**意图分类器**，负责将用户输入归类到预定义的意图中。

            ## 当前可用的意图列表

            %s

            ## 用户输入
            > %s

            ## 任务要求
            1. 从上述意图中选择**最匹配**的一个
            2. 如果用户输入与所有意图都不相关，返回 `intent_code = "UNKNOWN"`
            3. 给出 0.00-1.00 的置信度

            ## 输出格式（严格遵守 JSON，不含任何其他文本）
            ```json
            {
              "intent_code": "意图编码",
              "confidence": 0.95,
              "reasoning": "匹配理由（一句话）"
            }
            ```
            """.formatted(intentsBlock.toString(), userInput);
    }

    /**
     * 解析 LLM 返回的 JSON 结果.
     */
    private IntentResult parseIntentFromLLM(String llmResponse, List<Intent> intents) {
        try {
            // 提取 JSON 块（LLM 可能包裹在 ```json ... ``` 中）
            String json = extractJsonBlock(llmResponse);
            JsonNode node = objectMapper.readTree(json);

            String intentCode = node.path("intent_code").asText("UNKNOWN");
            double confidence = node.path("confidence").asDouble(0.5);
            String reasoning = node.path("reasoning").asText("");

            // 查找对应的意图定义
            Intent matchedIntent = intents.stream()
                    .filter(i -> i.getIntentCode().equals(intentCode))
                    .findFirst()
                    .orElse(null);

            if (matchedIntent == null || "UNKNOWN".equals(intentCode)) {
                return IntentResult.unknown(reasoning);
            }

            return IntentResult.builder()
                    .intentCode(matchedIntent.getIntentCode())
                    .intentName(matchedIntent.getIntentName())
                    .category(matchedIntent.getCategory())
                    .confidence(confidence)
                    .reasoning(reasoning)
                    .riskLevel(matchedIntent.getRiskLevel())
                    .requiredParams(matchedIntent.getRequiredParams())
                    .matchedLayer("LLM")
                    .build();

        } catch (Exception e) {
            log.error("[Intent] LLM 输出解析失败: response={}", llmResponse, e);
            return IntentResult.unknown("LLM 输出解析失败");
        }
    }

    /**
     * 从 LLM 输出中提取 JSON 块.
     */
    private String extractJsonBlock(String text) {
        // 处理 ```json ... ``` 包裹的情况
        if (text.contains("```json")) {
            int start = text.indexOf("```json") + 7;
            int end = text.indexOf("```", start);
            if (end > start) {
                return text.substring(start, end).trim();
            }
        }
        // 处理 ``` ... ``` 包裹的情况
        if (text.contains("```")) {
            int start = text.indexOf("```") + 3;
            int end = text.indexOf("```", start);
            if (end > start) {
                return text.substring(start, end).trim();
            }
        }
        return text.trim();
    }
}
```

### 6. 意图识别结果（Builder 模式）

```java
/**
 * 意图识别结果值对象 — Builder 模式构建.
 *
 * @author Agent Platform Team
 * @since 1.0.0
 */
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class IntentResult {

    /** 意图编码（如 ORDER_QUERY、LEAVE_APPLY、UNKNOWN） */
    private String intentCode;

    /** 意图名称 */
    private String intentName;

    /** 意图分类: FAQ / TASK / CHITCHAT / MULTI_STEP */
    private String category;

    /** 置信度 0.0-1.0 */
    private double confidence;

    /** 匹配理由 */
    private String reasoning;

    /** 风险等级: LOW / MEDIUM / HIGH / CRITICAL */
    private String riskLevel;

    /** 必要参数 Schema（JSON） */
    private List<Map<String, Object>> requiredParams;

    /** 匹配层: RULE / CACHE / LLM */
    private String matchedLayer;

    // ========== 工厂方法 ==========

    /**
     * 规则命中.
     */
    public static IntentResult matched(Intent intent, double confidence, String reasoning) {
        return IntentResult.builder()
                .intentCode(intent.getIntentCode())
                .intentName(intent.getIntentName())
                .category(intent.getCategory())
                .confidence(confidence)
                .reasoning(reasoning)
                .riskLevel(intent.getRiskLevel())
                .requiredParams(intent.getRequiredParams())
                .matchedLayer("RULE")
                .build();
    }

    /**
     * 未匹配到任何意图.
     */
    public static IntentResult unknown(String reason) {
        return IntentResult.builder()
                .intentCode("UNKNOWN")
                .intentName("未知意图")
                .category("UNKNOWN")
                .confidence(0.0)
                .reasoning(reason)
                .riskLevel("LOW")
                .matchedLayer("NONE")
                .build();
    }

    /**
     * 是否匹配成功.
     */
    public boolean isMatched() {
        return !"UNKNOWN".equals(intentCode);
    }

    /**
     * 是否为高风险意图.
     */
    public boolean isHighRisk() {
        return "HIGH".equals(riskLevel) || "CRITICAL".equals(riskLevel);
    }
}
```

### 7. 意图 CRUD 服务（Template Method + Observer 模式）

```java
/**
 * 意图应用服务 — Template Method 统一 CRUD 骨架.
 * <p>
 * 每个写操作后通过 Observer 模式通知缓存失效 + 重载识别规则.
 *
 * @author Agent Platform Team
 * @since 1.0.0
 */
@Service
@Slf4j
@RequiredArgsConstructor
public class IntentApplicationService {

    private final IntentRepository intentRepository;
    private final CacheRecognizer cacheRecognizer;
    private final IntentEventPublisher eventPublisher;

    /**
     * 创建意图 — Template Method 骨架.
     */
    @Transactional
    public IntentResponse createIntent(CreateIntentRequest request) {
        // 1. 校验
        validateIntentCodeUnique(request.getIntentCode());

        // 2. 构建
        Intent intent = Intent.builder()
                .intentId(IdGenerator.generate("int"))
                .tenantId(TenantContext.getCurrentTenantId())
                .intentCode(request.getIntentCode())
                .intentName(request.getIntentName())
                .category(request.getCategory())
                .patterns(request.getPatterns())
                .examples(request.getExamples())
                .llmPrompt(request.getLlmPrompt())
                .requiredParams(request.getRequiredParams())
                .riskLevel(request.getRiskLevel())
                .status(IntentStatus.ACTIVE)
                .build();

        // 3. 持久化
        intentRepository.save(intent);

        // 4. Observer: 通知缓存失效
        eventPublisher.publishIntentCreated(intent);

        log.info("[Intent] 创建意图: code={}, name={}, category={}",
                intent.getIntentCode(), intent.getIntentName(), intent.getCategory());

        return IntentAssembler.toResponse(intent);
    }

    /**
     * 编辑意图.
     */
    @Transactional
    public IntentResponse updateIntent(String intentId, UpdateIntentRequest request) {
        Intent intent = intentRepository.findByIntentId(intentId)
                .orElseThrow(() -> new ResourceNotFoundException("意图", intentId));

        // 更新字段
        intent.updateDefinition(
                request.getIntentName(),
                request.getPatterns(),
                request.getExamples(),
                request.getLlmPrompt(),
                request.getRequiredParams(),
                request.getRiskLevel()
        );

        intentRepository.update(intent);

        // Observer: 通知缓存失效 + 规则重载
        eventPublisher.publishIntentUpdated(intent);

        return IntentAssembler.toResponse(intent);
    }

    /**
     * 启停意图.
     */
    @Transactional
    public void toggleStatus(String intentId, IntentStatus newStatus) {
        Intent intent = intentRepository.findByIntentId(intentId)
                .orElseThrow(() -> new ResourceNotFoundException("意图", intentId));

        intent.setStatus(newStatus);
        intentRepository.updateStatus(intentId, newStatus);

        // Observer: 通知缓存失效 + 规则重载
        eventPublisher.publishIntentStatusChanged(intent, newStatus);
    }

    /**
     * 测试意图识别 — Command 模式.
     * <p>
     * 输入文本，返回三步识别的详细过程（调试用）.
     */
    public IntentTestResponse testRecognition(String intentId, String testInput) {
        Intent intent = intentRepository.findByIntentId(intentId)
                .orElseThrow(() -> new ResourceNotFoundException("意图", intentId));

        long startTime = System.nanoTime();
        IntentResult result = recognitionChain.recognize(testInput);
        long costNanos = System.nanoTime() - startTime;

        return IntentTestResponse.builder()
                .input(testInput)
                .expectedIntentCode(intent.getIntentCode())
                .actualIntentCode(result.getIntentCode())
                .matched(result.getIntentCode().equals(intent.getIntentCode()))
                .confidence(result.getConfidence())
                .matchedLayer(result.getMatchedLayer())
                .reasoning(result.getReasoning())
                .costMs(costNanos / 1_000_000.0)
                .build();
    }

    /**
     * 批量测试 — 输入 CSV（文本,期望意图），返回准确率.
     */
    public BatchTestResponse batchTest(List<TestItem> items) {
        int total = items.size();
        int correct = 0;
        List<IntentTestResponse> details = new ArrayList<>();

        for (TestItem item : items) {
            IntentResult result = recognitionChain.recognize(item.getInput());
            boolean matched = result.getIntentCode().equals(item.getExpectedIntentCode());
            if (matched) correct++;
            details.add(IntentTestResponse.builder()
                    .input(item.getInput())
                    .expectedIntentCode(item.getExpectedIntentCode())
                    .actualIntentCode(result.getIntentCode())
                    .matched(matched)
                    .confidence(result.getConfidence())
                    .build());
        }

        return BatchTestResponse.builder()
                .total(total)
                .correct(correct)
                .accuracy((double) correct / total)
                .details(details)
                .build();
    }

    private void validateIntentCodeUnique(String intentCode) {
        String tenantId = TenantContext.getCurrentTenantId();
        if (intentRepository.existsByTenantAndCode(tenantId, intentCode)) {
            throw new BusinessException(400, "意图编码已存在: " + intentCode);
        }
    }
}

/**
 * 意图变更监听器 — Observer 模式 Observer.
 * <p>
 * 异步处理：缓存失效 + 规则热重载.
 */
@Component
@Slf4j
@RequiredArgsConstructor
public class IntentChangedListener {

    private final CacheRecognizer cacheRecognizer;

    @EventListener
    @Async
    public void onIntentChanged(IntentChangedEvent event) {
        Intent intent = event.getIntent();
        // 失效该租户的所有意图缓存
        cacheRecognizer.evictCache(intent.getTenantId());
        log.info("[Intent] 缓存已失效: tenantId={}, action={}",
                intent.getTenantId(), event.getAction());
    }

    @EventListener
    @Async
    public void onIntentStatusChanged(IntentStatusChangedEvent event) {
        cacheRecognizer.evictCache(event.getIntent().getTenantId());
        log.info("[Intent] 状态变更缓存失效: code={}, status={}",
                event.getIntent().getIntentCode(), event.getNewStatus());
    }
}

## 关键设计决策

1. **Chain of Responsibility 三层识别** — 规则（1ms）→ 缓存（5ms）→ LLM（500ms），命中即短路，最佳性能路径
2. **Strategy 模式封装匹配算法** — 正则匹配、关键词加权、LLM 分类三种策略可独立替换，新策略只需实现接口
3. **Template Method 定义 LLM 分类骨架** — 加载→构建 Prompt→调用→解析 四步，子类可覆盖 buildPrompt() 切换 Few-shot / CoT 策略
4. **Observer 模式解耦意图变更** — 意图增删改后 Spring Event 异步通知缓存失效，不影响请求响应时间
5. **Builder 模式构建 IntentResult** — 10+ 字段的结果对象，Builder 确保不可变性和构建正确性
6. **LLM 低温度（0.1）增强确定性** — 意图分类属于确定性任务，低温度减少随机性
7. **意图测试 API 支持调试** — `POST /intents/{id}/test` 返回三步识别的详细过程，方便运维排查误识别
