# 对话状态机与记忆管理

## 所属阶段
**P1 核心能力 → T3 意图识别与对话管理**

## 使用技术
- Spring StateMachine（对话状态流转）
- Redis（短期/会话级记忆，TTL 30 分钟）
- MySQL + MyBatis（长期记忆持久化）
- Spring AI（LLM 驱动记忆提取 + 摘要）
- Spring `@Async`（异步记忆写入，不阻塞对话）

## 涉及数据库表
- `t_long_term_memory` — 长期记忆表
- `t_message` — 关联消息源
- `t_conversation` — 关联会话

## 设计模式应用

| 设计模式 | 应用场景 | 核心价值 |
|----------|----------|----------|
| **State** | 对话状态机（INIT→AWAIT_PARAM→EXECUTING→CONFIRM→DONE） | 状态转移显式化，禁止非法转移 |
| **Memento** | 对话上下文快照（检查点/回滚） | 保存与恢复对话状态，支持断点续传 |
| **Strategy** | 记忆提取策略（LLM 摘要 / 正则提取 / 模板匹配） | 不同记忆类型不同提取算法 |
| **Template Method** | 记忆生命周期（加载→使用→更新→提取→持久化） | 统一生命周期，子类填差异 |
| **Observer** | 对话状态变更通知（WebSocket 推送执行进度） | 解耦状态变更与 UI 更新 |
| **Builder** | LongTermMemory 实体构建 | 8 字段实体，链式构建 |
| **Factory** | 记忆提取器工厂（按 memory_type 分配提取器） | 隔离创建逻辑，新增类型不改工厂 |
| **Repository** | 记忆仓储抽象（Redis 短期 + MySQL 长期） | 领域层不感知存储细节 |

### 1. 对话状态机总览（State 模式）

```
                          ┌──────────┐
        createConversation │   INIT   │ ← 会话创建
                          └────┬─────┘
                               │ 用户发送第一条消息
                               ▼
                          ┌──────────┐
                    ┌────→│AWAIT_PARAM│ ← 等待参数补充（意图识别后缺必需参数）
                    │     └────┬─────┘
                    │          │ 参数完整 / 用户确认
                    │          ▼
                    │     ┌──────────┐
                    │     │EXECUTING │ ← 执行中（调 LLM / 工具 / 检索）
                    │     └────┬─────┘
                    │          │ 需要人工确认?
                    │  ┌───────┼───────┐
                    │  │       │       │
                    │  ▼       ▼       ▼
                    │ ┌────┐ ┌────┐ ┌─────┐
                    │ │CONFIRM│ │DONE│ │ERROR│
                    │ └──┬─┘ └────┘ └─────┘
                    │    │ 用户同意
                    │    └──→ back to EXECUTING
                    │    用户拒绝
                    └────── back to INIT
```

### 2. Spring StateMachine 配置（State 模式核心）

```java
/**
 * 对话状态机配置 — State 模式实现.
 * <p>
 * 使用 Spring StateMachine 管理对话生命周期状态转移，
 * 每个状态转移都有 Guard（前置条件）和 Action（副作用）.
 *
 * @author Agent Platform Team
 * @since 1.0.0
 */
@Configuration
@EnableStateMachineFactory
public class ConversationStateMachineConfig
        extends EnumStateMachineConfigurerAdapter<ConversationState, ConversationEvent> {

    /**
     * 状态定义.
     */
    @Override
    public void configure(StateMachineStateConfigurer<ConversationState, ConversationEvent> states)
            throws Exception {
        states.withStates()
                .initial(ConversationState.INIT)
                .state(ConversationState.AWAIT_PARAM)
                .state(ConversationState.EXECUTING)
                .state(ConversationState.CONFIRMATION)
                .end(ConversationState.DONE)
                .end(ConversationState.ERROR);
    }

    /**
     * 状态转移规则 — 每个转移都有 source → target + event + guard.
     */
    @Override
    public void configure(StateMachineTransitionConfigurer<ConversationState, ConversationEvent> transitions)
            throws Exception {
        transitions
            // INIT → AWAIT_PARAM: 收到消息
            .withExternal()
                .source(ConversationState.INIT)
                .target(ConversationState.AWAIT_PARAM)
                .event(ConversationEvent.MESSAGE_RECEIVED)
                .and()
            // AWAIT_PARAM → EXECUTING: 参数完整
            .withExternal()
                .source(ConversationState.AWAIT_PARAM)
                .target(ConversationState.EXECUTING)
                .event(ConversationEvent.PARAMS_COMPLETE)
                .guard(paramsCompleteGuard())
                .and()
            // AWAIT_PARAM → AWAIT_PARAM: 参数仍不完整（自循环）
            .withInternal()
                .source(ConversationState.AWAIT_PARAM)
                .event(ConversationEvent.MESSAGE_RECEIVED)
                .guard(paramsIncompleteGuard())
                .action(askForMissingParamsAction())
                .and()
            // EXECUTING → DONE: 执行完成
            .withExternal()
                .source(ConversationState.EXECUTING)
                .target(ConversationState.DONE)
                .event(ConversationEvent.COMPLETED)
                .and()
            // EXECUTING → CONFIRMATION: 需要人工确认
            .withExternal()
                .source(ConversationState.EXECUTING)
                .target(ConversationState.CONFIRMATION)
                .event(ConversationEvent.CONFIRMATION_NEEDED)
                .and()
            // CONFIRMATION → EXECUTING: 用户确认，继续执行
            .withExternal()
                .source(ConversationState.CONFIRMATION)
                .target(ConversationState.EXECUTING)
                .event(ConversationEvent.USER_CONFIRMED)
                .and()
            // CONFIRMATION → DONE: 用户拒绝
            .withExternal()
                .source(ConversationState.CONFIRMATION)
                .target(ConversationState.DONE)
                .event(ConversationEvent.USER_REJECTED)
                .and()
            // 任意状态 → ERROR: 异常发生
            .withExternal()
                .source(ConversationState.INIT)
                .target(ConversationState.ERROR)
                .event(ConversationEvent.ERROR_OCCURRED)
                .and()
            .withExternal()
                .source(ConversationState.AWAIT_PARAM)
                .target(ConversationState.ERROR)
                .event(ConversationEvent.ERROR_OCCURRED)
                .and()
            .withExternal()
                .source(ConversationState.EXECUTING)
                .target(ConversationState.ERROR)
                .event(ConversationEvent.ERROR_OCCURRED);
    }
}
```

### 3. 对话状态与事件枚举

```java
/**
 * 对话状态枚举 — State 模式的 State.
 */
public enum ConversationState {

    /** 初始状态 — 会话刚创建，等待首次用户输入 */
    INIT("初始"),

    /** 等待参数 — 意图已识别但缺少必要参数 */
    AWAIT_PARAM("等待参数补充"),

    /** 执行中 — 调用 LLM / 工具 / 检索 */
    EXECUTING("执行中"),

    /** 等待确认 — 高风险操作需人工审批 */
    CONFIRMATION("等待确认"),

    /** 完成 — 本次对话回合正常结束 */
    DONE("完成"),

    /** 异常 — 执行过程中发生错误 */
    ERROR("异常");

    @Getter
    private final String label;

    ConversationState(String label) { this.label = label; }
}

/**
 * 对话事件枚举 — State 模式的 Event.
 */
public enum ConversationEvent {
    /** 收到用户消息 */
    MESSAGE_RECEIVED,
    /** 参数收集完整 */
    PARAMS_COMPLETE,
    /** 参数仍不完整 */
    PARAMS_INCOMPLETE,
    /** 执行完成 */
    COMPLETED,
    /** 需要人工确认 */
    CONFIRMATION_NEEDED,
    /** 用户确认 */
    USER_CONFIRMED,
    /** 用户拒绝 */
    USER_REJECTED,
    /** 异常发生 */
    ERROR_OCCURRED
}

### 4. 状态机 Guard 与 Action

```java
/**
 * 状态机 Guard — 参数完整性校验.
 */
@Component
public class ConversationStateGuards {

    /**
     * Guard: 参数是否完整.
     */
    @Bean
    public Guard<ConversationState, ConversationEvent> paramsCompleteGuard() {
        return context -> {
            IntentResult intent = context.getExtendedState()
                    .get("currentIntent", IntentResult.class);
            Map<String, Object> collectedParams = context.getExtendedState()
                    .get("collectedParams", Map.class);

            if (intent == null || intent.getRequiredParams() == null) {
                return true;
            }
            // 所有 required_params 的 key 都已收集到
            return intent.getRequiredParams().stream()
                    .allMatch(param -> collectedParams.containsKey(param.get("name")));
        };
    }

    /**
     * Guard: 参数不完整.
     */
    @Bean
    public Guard<ConversationState, ConversationEvent> paramsIncompleteGuard() {
        return context -> {
            IntentResult intent = context.getExtendedState()
                    .get("currentIntent", IntentResult.class);
            Map<String, Object> collectedParams = context.getExtendedState()
                    .get("collectedParams", Map.class);

            if (intent == null || intent.getRequiredParams() == null) {
                return false;
            }
            return intent.getRequiredParams().stream()
                    .anyMatch(param -> !collectedParams.containsKey(param.get("name")));
        };
    }
}

/**
 * 状态机 Action — 询问缺失参数.
 */
@Component
public class ConversationStateActions {

    @Bean
    public Action<ConversationState, ConversationEvent> askForMissingParamsAction() {
        return context -> {
            IntentResult intent = context.getExtendedState()
                    .get("currentIntent", IntentResult.class);
            Map<String, Object> collectedParams = context.getExtendedState()
                    .get("collectedParams", Map.class);

            // 找出第一个缺失的参数，生成询问消息
            Map<String, Object> missingParam = intent.getRequiredParams().stream()
                    .filter(p -> !collectedParams.containsKey(p.get("name")))
                    .findFirst()
                    .orElse(null);

            if (missingParam != null) {
                String prompt = "请提供" + missingParam.get("description");
                context.getExtendedState().getVariables().put("askPrompt", prompt);
            }
        };
    }
}
```

### 5. 对话上下文管理（Memento 模式）

```java
/**
 * 对话上下文 — Memento 模式保存/恢复对话快照.
 * <p>
 * 对话执行过程中可创建检查点（checkpoint），
 * 异常/中断时可回滚到最近的检查点继续执行.
 *
 * @author Agent Platform Team
 * @since 1.0.0
 */
@Data
@Builder
public class DialogContext {

    /** 会话 ID */
    private String conversationId;

    /** 当前状态 */
    private ConversationState currentState;

    /** 识别到的意图 */
    private IntentResult currentIntent;

    /** 已收集的参数 */
    private Map<String, Object> collectedParams;

    /** 当前执行步骤索引 */
    private int currentStepIndex;

    /** 已完成步骤的返回值 */
    private List<Map<String, Object>> stepResults;

    /** 上下文变量（可跨步骤共享） */
    private Map<String, Object> variables;

    // ========== Memento 模式 ==========

    /**
     * 创建快照（Memento）.
     */
    public DialogContextMemento createSnapshot() {
        return DialogContextMemento.builder()
                .conversationId(this.conversationId)
                .currentState(this.currentState)
                .currentIntent(this.currentIntent)
                .collectedParams(new HashMap<>(this.collectedParams))
                .currentStepIndex(this.currentStepIndex)
                .stepResults(new ArrayList<>(this.stepResults))
                .variables(new HashMap<>(this.variables))
                .snapshotTime(System.currentTimeMillis())
                .build();
    }

    /**
     * 从快照恢复.
     */
    public static DialogContext restore(DialogContextMemento memento) {
        return DialogContext.builder()
                .conversationId(memento.getConversationId())
                .currentState(memento.getCurrentState())
                .currentIntent(memento.getCurrentIntent())
                .collectedParams(memento.getCollectedParams())
                .currentStepIndex(memento.getCurrentStepIndex())
                .stepResults(memento.getStepResults())
                .variables(memento.getVariables())
                .build();
    }
}

/**
 * 对话上下文快照 — Memento 模式的值对象.
 * <p>
 * 不可变（创建后不修改），记录某一时刻的完整对话状态.
 */
@Data
@Builder
public class DialogContextMemento {

    private String conversationId;
    private ConversationState currentState;
    private IntentResult currentIntent;
    private Map<String, Object> collectedParams;
    private int currentStepIndex;
    private List<Map<String, Object>> stepResults;
    private Map<String, Object> variables;

    /** 快照时间戳 */
    private long snapshotTime;
}

/**
 * 对话上下文管理器 — Caretaker 角色.
 * <p>
 * 管理 Memento 的生命周期：创建、存储、恢复、清理.
 */
@Component
@Slf4j
public class DialogContextManager {

    /** 每个会话最多保留 5 个检查点 */
    private static final int MAX_CHECKPOINTS = 5;

    private final RedisTemplate<String, String> redisTemplate;
    private final ObjectMapper objectMapper;

    /**
     * 保存上下文快照到 Redis.
     */
    public void saveCheckpoint(String conversationId, DialogContext context) {
        DialogContextMemento memento = context.createSnapshot();
        String key = "checkpoint:" + conversationId;
        try {
            String json = objectMapper.writeValueAsString(memento);
            redisTemplate.opsForList().leftPush(key, json);
            redisTemplate.opsForList().trim(key, 0, MAX_CHECKPOINTS - 1);
            redisTemplate.expire(key, Duration.ofHours(1));
            log.debug("[Dialog] 检查点保存: convId={}, state={}, step={}",
                    conversationId, memento.getCurrentState(), memento.getCurrentStepIndex());
        } catch (JsonProcessingException e) {
            log.error("[Dialog] 检查点序列化失败", e);
        }
    }

    /**
     * 从最近检查点恢复.
     */
    public Optional<DialogContext> restoreLatest(String conversationId) {
        String key = "checkpoint:" + conversationId;
        String json = redisTemplate.opsForList().index(key, 0);
        if (json != null) {
            try {
                DialogContextMemento memento = objectMapper.readValue(json, DialogContextMemento.class);
                log.info("[Dialog] 从检查点恢复: convId={}, state={}, step={}",
                        conversationId, memento.getCurrentState(), memento.getCurrentStepIndex());
                return Optional.of(DialogContext.restore(memento));
            } catch (JsonProcessingException e) {
                log.error("[Dialog] 检查点反序列化失败", e);
            }
        }
        return Optional.empty();
    }

    /**
     * 清理检查点（对话结束后）.
     */
    public void clearCheckpoints(String conversationId) {
        redisTemplate.delete("checkpoint:" + conversationId);
        log.debug("[Dialog] 检查点清理: convId={}", conversationId);
    }
}
```

### 6. 短期记忆管理（Redis — Strategy 模式）

```java
/**
 * 会话短期记忆服务 — 管理 Redis 中最近 N 轮对话.
 * <p>
 * 每次交互后刷新 TTL，确保活跃对话的上下文始终可用.
 *
 * @author Agent Platform Team
 * @since 1.0.0
 */
@Service
@Slf4j
@RequiredArgsConstructor
public class SessionMemoryService {

    /** 保留最近轮数 */
    private static final int MAX_ROUNDS = 20;
    /** 每轮 2 条（user + assistant） */
    private static final int MAX_MESSAGES = MAX_ROUNDS * 2;
    /** 过期时间 */
    private static final Duration TTL = Duration.ofMinutes(30);

    private final RedisTemplate<String, String> redisTemplate;
    private final ObjectMapper objectMapper;

    /**
     * 追加消息到短期记忆.
     * <p>
     * 使用 Redis List，RIGHT PUSH + TRIM 保持最新 N 条.
     */
    public void appendMessage(String conversationId, Message message) {
        String key = buildSessionKey(conversationId);
        try {
            String json = objectMapper.writeValueAsString(MessageCacheEntry.from(message));
            redisTemplate.opsForList().rightPush(key, json);
            redisTemplate.opsForList().trim(key, -MAX_MESSAGES, -1);
            redisTemplate.expire(key, TTL);
        } catch (JsonProcessingException e) {
            log.warn("[SessionMemory] 序列化失败: msgId={}", message.getMessageId(), e);
        }
    }

    /**
     * 获取最近 rounds 轮对话（Strategy 模式选择加载策略）.
     *
     * @param conversationId 会话 ID
     * @param rounds 加载最近 N 轮
     * @return 消息列表（按时间正序）
     */
    public List<Message> getRecentMessages(String conversationId, int rounds) {
        String key = buildSessionKey(conversationId);
        int count = Math.min(rounds * 2, MAX_MESSAGES);
        List<String> entries = redisTemplate.opsForList().range(key, -count, -1);
        if (entries == null || entries.isEmpty()) {
            return List.of();
        }
        return entries.stream()
                .map(this::deserializeMessage)
                .filter(Objects::nonNull)
                .toList();
    }

    /**
     * 加载完整会话上下文供 LLM 使用.
     * <p>
     * Strategy: 根据 Token 预算智能选择加载轮数.
     *
     * @param maxTokens LLM 上下文可用 Token 数
     */
    public List<Message> loadContextForLLM(String conversationId, int maxTokens) {
        // 从最新消息开始加载，累计 Token 不超预算
        List<Message> allMessages = getRecentMessages(conversationId, MAX_ROUNDS);
        List<Message> result = new ArrayList<>();
        int accumulatedTokens = 0;

        // 倒序遍历（从最新到最旧）
        for (int i = allMessages.size() - 1; i >= 0; i--) {
            Message msg = allMessages.get(i);
            int msgTokens = msg.getTokenCount() != null ? msg.getTokenCount() : 0;
            if (accumulatedTokens + msgTokens > maxTokens) {
                break;
            }
            result.add(0, msg);  // 保持正序
            accumulatedTokens += msgTokens;
        }

        log.debug("[SessionMemory] 加载上下文: convId={}, rounds={}, tokens={}/{}",
                conversationId, result.size() / 2, accumulatedTokens, maxTokens);
        return result;
    }

    /**
     * 刷新 TTL（每次用户交互时调用）.
     */
    public void touch(String conversationId) {
        redisTemplate.expire(buildSessionKey(conversationId), TTL);
    }

    /**
     * 结束会话时清理短期记忆.
     */
    public void clear(String conversationId) {
        redisTemplate.delete(buildSessionKey(conversationId));
        log.debug("[SessionMemory] 清理: convId={}", conversationId);
    }

    private String buildSessionKey(String conversationId) {
        return "session:" + TenantContext.getCurrentTenantId() + ":" + conversationId;
    }

    private Message deserializeMessage(String json) {
        try {
            return objectMapper.readValue(json, MessageCacheEntry.class).toMessage();
        } catch (JsonProcessingException e) {
            log.warn("[SessionMemory] 反序列化失败", e);
            return null;
        }
    }
}
```

### 7. 长期记忆管理（MySQL — Decorator + Factory 模式）

```java
/**
 * 长期记忆应用服务 — 管理跨会话的用户记忆.
 * <p>
 * 异步提取（不阻塞对话），支持 FACT / PREFERENCE / CONTEXT / SUMMARY 四种类型.
 *
 * @author Agent Platform Team
 * @since 1.0.0
 */
@Service
@Slf4j
@RequiredArgsConstructor
public class LongTermMemoryService {

    private final LongTermMemoryRepository memoryRepository;
    private final SessionMemoryService sessionMemoryService;
    private final ChatClient chatClient;
    private final ObjectMapper objectMapper;

    /** 记忆提取器工厂 — Factory 模式 */
    private final Map<MemoryType, MemoryExtractor> extractorMap;

    /**
     * 对话结束后异步提取关键信息.
     * <p>
     * Template Method:
     * 1. 加载完整对话历史
     * 2. LLM 摘要 + 提取事实
     * 3. 解析结构化记忆
     * 4. 按 memory_type 分配提取器（Factory 模式）
     * 5. UPSERT 到 MySQL（key 相同则更新置信度）
     */
    @Async
    public void extractAndSave(String conversationId, String userId) {
        log.info("[LTM] 开始提取: convId={}, userId={}", conversationId, userId);

        try {
            // Step 1: 加载对话历史
            List<Message> history = sessionMemoryService.getRecentMessages(conversationId, MAX_ROUNDS);

            if (history.size() < 4) {
                log.debug("[LTM] 对话轮次不足，跳过: convId={}", conversationId);
                return;
            }

            // Step 2: LLM 提取关键信息
            String summary = summarizeConversation(history);
            List<MemoryFact> facts = parseFacts(summary, conversationId);

            // Step 3-4: Factory 模式分配提取器 + 持久化
            String tenantId = TenantContext.getCurrentTenantId();
            for (MemoryFact fact : facts) {
                MemoryType type = fact.getType();
                MemoryExtractor extractor = extractorMap.getOrDefault(
                        type, extractorMap.get(MemoryType.FACT));

                List<LongTermMemory> memories = extractor.extract(fact, history);
                for (LongTermMemory memory : memories) {
                    memory.setTenantId(tenantId);
                    memory.setUserId(userId);
                    memory.setSource(conversationId);
                    memoryRepository.upsert(memory);  // key 相同则更新
                }
            }

            log.info("[LTM] 提取完成: convId={}, facts={}, memories={}",
                    conversationId, facts.size(),
                    facts.stream().mapToInt(f -> extractorMap.getOrDefault(f.getType(),
                            extractorMap.get(MemoryType.FACT)).extract(f, history).size()).sum());

        } catch (Exception e) {
            log.error("[LTM] 提取失败: convId={}", conversationId, e);
        }
    }

    /**
     * 新会话开始时加载用户长期记忆.
     * <p>
     * Strategy: 按记忆类型优先级加载（PREFERENCE > CONTEXT > FACT > SUMMARY）
     */
    public List<LongTermMemory> loadUserMemories(String userId) {
        String tenantId = TenantContext.getCurrentTenantId();
        List<LongTermMemory> memories = memoryRepository.findByUserId(tenantId, userId);

        // 过滤已过期的记忆
        memories = memories.stream()
                .filter(m -> m.getExpireAt() == null || m.getExpireAt().isAfter(LocalDateTime.now()))
                .sorted(Comparator.comparingInt(m -> m.getMemoryType().getLoadPriority()))
                .toList();

        log.debug("[LTM] 加载用户记忆: userId={}, count={}", userId, memories.size());
        return memories;
    }

    /**
     * 构建系统提示词时注入长期记忆.
     */
    public String injectMemoriesIntoPrompt(String systemPrompt, List<LongTermMemory> memories) {
        if (memories.isEmpty()) {
            return systemPrompt;
        }
        StringBuilder memoryBlock = new StringBuilder("\n\n## 关于该用户的已知信息\n");
        for (LongTermMemory memory : memories) {
            memoryBlock.append("- ").append(memory.getMemoryKey())
                    .append(": ").append(memory.getMemoryValue())
                    .append(" (置信度: ").append(String.format("%.0f%%", memory.getConfidence() * 100))
                    .append(")\n");
        }
        return systemPrompt + memoryBlock;
    }

    /**
     * LLM 摘要对话并提取关键事实.
     */
    private String summarizeConversation(List<Message> history) {
        String conversationText = history.stream()
                .map(m -> m.getRole().getLabel() + ": " + m.getContent())
                .collect(Collectors.joining("\n"));

        return chatClient.prompt()
                .user("""
                    从以下对话中提取用户的关键信息和偏好。返回 JSON 格式。

                    ## 对话内容
                    %s

                    ## 提取规则
                    1. FACT: 用户明确陈述的事实（姓名、职位、公司等）
                    2. PREFERENCE: 用户表达的偏好（喜欢的回复风格、格式等）
                    3. CONTEXT: 当前对话的重要上下文
                    4. SUMMARY: 对话的简短摘要（1-2 句话）

                    ## 输出格式
                    ```json
                    {
                      "facts": [
                        {"type": "FACT", "key": "user_name", "value": "张三", "confidence": 0.95},
                        {"type": "PREFERENCE", "key": "preferred_language", "value": "中文", "confidence": 0.8}
                      ],
                      "summary": "用户咨询了订单状态并表达了不满..."
                    }
                    ```
                    """.formatted(conversationText))
                .options(OpenAiChatOptions.builder()
                        .withTemperature(0.1)
                        .withMaxTokens(1024)
                        .build())
                .call()
                .content();
    }

    /**
     * 解析 LLM 提取的记忆事实.
     */
    private List<MemoryFact> parseFacts(String llmResponse, String conversationId) {
        try {
            String json = extractJsonBlock(llmResponse);
            JsonNode root = objectMapper.readTree(json);
            List<MemoryFact> facts = new ArrayList<>();
            for (JsonNode node : root.path("facts")) {
                facts.add(MemoryFact.builder()
                        .type(MemoryType.valueOf(node.path("type").asText("FACT")))
                        .key(node.path("key").asText())
                        .value(node.path("value").asText())
                        .confidence(node.path("confidence").asDouble(0.8))
                        .build());
            }
            return facts;
        } catch (Exception e) {
            log.error("[LTM] 解析 LLM 输出失败", e);
            return List.of();
        }
    }

    private String extractJsonBlock(String text) {
        if (text.contains("```json")) {
            int start = text.indexOf("```json") + 7;
            int end = text.indexOf("```", start);
            return end > start ? text.substring(start, end).trim() : text;
        }
        return text;
    }
}

public interface MemoryExtractor {

    /** 支持的记忆类型 */
    MemoryType supportedType();

    /**
     * 从对话历史中提取记忆.

     * @param fact    识别的原始事实

/**
 * 事实提取器 — 提取用户明确陈述的事实.
 */
@Component
public class FactMemoryExtractor implements MemoryExtractor {

    @Override
    public MemoryType supportedType() { return MemoryType.FACT; }

    @Override
    public List<LongTermMemory> extract(MemoryFact fact, List<Message> history) {
        // FACT 类型记忆永不过期（除非被更高置信度的记录覆盖）
        return List.of(LongTermMemory.builder()
                .memoryType(MemoryType.FACT)

    @Override

    public MemoryExtractorRegistry(List<MemoryExtractor> extractors) {

## 关键设计决策

1. **State 模式管理对话状态** — Spring StateMachine + Guard/Action 实现显式状态转移，禁止非法路径
2. **Memento 模式实现断点续传** — 对话执行到一半出错/中断时，从 Redis 检查点恢复
3. **Strategy 模式区分记忆提取算法** — FACT/PREFERENCE/CONTEXT/SUMMARY 四种提取器独立演进
4. **Factory 模式自动装配提取器** — Spring 注入所有 MemoryExtractor 实现，按 memory_type 路由
5. **Template Method 定义记忆生命周期** — 加载→使用→更新→提取→持久化 五步统一流程
6. **短期记忆 Redis List** — RIGHT PUSH + TRIM 保持最新 40 条（20轮），TTL 30 分钟自动淘汰
7. **长期记忆 UPSERT** — key 相同则更新（置信度取最大值），避免重复记录
8. **异步记忆提取不阻塞** — `@Async` + LLM 调用在后台执行，对话主流程不受影响
9. **记忆注入 Prompt** — 新会话开始时自动将用户长期记忆拼入 system prompt

## 异常处理

| 场景 | 行为 |
|------|------|
| LLM 记忆提取失败 | 记录错误日志，不阻塞对话流程 |
| Redis 短期记忆不可用 | 降级为仅使用 DB 历史消息 |
| MySQL 长期记忆写入失败 | 记录错误，下次对话时重试提取 |
| 记忆过期 | 定时任务清理 + 加载时过滤 |
| 检查点反序列化失败 | 从 INIT 状态重新开始（清空检查点） |
