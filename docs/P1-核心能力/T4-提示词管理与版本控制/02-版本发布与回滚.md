# 提示词版本发布与回滚

## 所属阶段
**P1 核心能力 → T4 提示词管理与版本控制**

## 使用技术
- MyBatis-Plus
- Redis（缓存失效通知）

## 涉及数据库表
- `t_prompt_template`（主表），`t_prompt_template_version`（历史版本）

## API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/prompts/{id}/publish` | 发布当前草稿（版本号 +1） |
| GET | `/api/v1/prompts/{id}/versions` | 查看版本历史列表 |
| GET | `/api/v1/prompts/{id}/versions/{version}` | 查看指定版本内容 |
| POST | `/api/v1/prompts/{id}/rollback?version=N` | 回滚到指定版本 |
| POST | `/api/v1/prompts/{id}/diff?v1=N&v2=M` | 两个版本的差异对比 |

## 实现方案

### 1. 发布流程

```java
@Service
@Transactional
public class PromptPublishService {

    public PromptTemplate publish(String promptId) {
        PromptTemplate current = promptRepository.findByPromptId(promptId);
        if (!"DRAFT".equals(current.getStatus())) {
            throw new BusinessException(400, "只有草稿状态可发布");
        }

        int newVersion = current.getVersion() + 1;

        // 1. 将当前版本存入历史表
        PromptTemplateVersion history = PromptTemplateVersion.builder()
                .promptId(promptId)
                .version(newVersion)
                .templateText(current.getTemplateText())
                .variables(current.getVariables())
                .changeLog("发布 v" + newVersion)
                .publisher(TenantContext.getCurrentUserId())
                .publishedAt(LocalDateTime.now())
                .build();
        versionRepository.save(history);

        // 2. 更新主表版本号和状态
        current.setVersion(newVersion);
        current.setStatus("PUBLISHED");
        promptRepository.updateById(current);

        // 3. Redis 缓存失效通知
        redisTemplate.convertAndSend("prompt:invalidate", promptId);

        return current;
    }
}
```

### 2. 回滚流程

```java
public PromptTemplate rollback(String promptId, int targetVersion) {
    // 1. 读取目标版本
    PromptTemplateVersion target = versionRepository.findByPromptAndVersion(promptId, targetVersion);
    if (target == null) {
        throw new ResourceNotFoundException("版本", promptId + " v" + targetVersion);
    }

    // 2. 目标版本内容作为新草稿
    PromptTemplate current = promptRepository.findByPromptId(promptId);
    int newVersion = current.getVersion() + 1;

    // 3. 创建新版本记录（回滚也是新版本）
    PromptTemplateVersion rollbackVersion = PromptTemplateVersion.builder()
            .promptId(promptId)
            .version(newVersion)
            .templateText(target.getTemplateText())
            .variables(target.getVariables())
            .changeLog("回滚至 v" + targetVersion)
            .publisher(TenantContext.getCurrentUserId())
            .publishedAt(LocalDateTime.now())
            .build();
    versionRepository.save(rollbackVersion);

    // 4. 更新主表
    current.setTemplateText(target.getTemplateText());
    current.setVariables(target.getVariables());
    current.setVersion(newVersion);
    current.setStatus("PUBLISHED");
    promptRepository.updateById(current);

    // 5. 缓存失效
    redisTemplate.convertAndSend("prompt:invalidate", promptId);

    return current;
}
```

### 3. 缓存策略

```java
@Component
public class CachedPromptRepository {

    @Cacheable(value = "prompt", key = "#promptId")
    public PromptTemplate getByPromptId(String promptId) {
        return promptRepository.findByPromptId(promptId);
    }

    // 监听 Redis Pub/Sub，收到失效通知时清除本地缓存
    @RedisMessageListener(topic = "prompt:invalidate")
    public void onInvalidate(String promptId) {
        cacheManager.getCache("prompt").evict(promptId);
        log.info("提示词缓存已失效: {}", promptId);
    }
}
```
