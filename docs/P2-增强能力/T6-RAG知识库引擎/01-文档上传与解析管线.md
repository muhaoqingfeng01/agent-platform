# 文档上传与解析管线

## 所属阶段
**P2 增强能力 → T6 RAG 知识库引擎**

## 使用技术
- Apache Tika（文档解析）
- MinIO（对象存储）
- Spring `@Async`（异步处理）

## 涉及数据库表
- `t_document`, `t_knowledge_base`

## API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/knowledge-bases/{id}/documents` | 上传文档（multipart/form-data） |
| GET | `/api/v1/documents` | 文档列表 |
| GET | `/api/v1/documents/{id}` | 文档详情 |
| GET | `/api/v1/documents/{id}/status` | 查询处理状态 |
| DELETE | `/api/v1/documents/{id}` | 删除文档（含 MinIO + Milvus） |

## 实现方案

### 1. 文档处理管线

```
上传 → MinIO 存储 → Tika 解析纯文本 → 语义切分 chunk → Embedding 向量化
  │        │              │                  │                   │
  │  UPLOADING  →   PARSING   →   CHUNKING   →   EMBEDDING   →   READY
  └────────────────────────────────────────────────────────────────┘
                          状态流转
```

### 2. DocumentPipelineService

```java
@Service
@Slf4j
public class DocumentPipelineService {

    private final MinioClient minioClient;
    private final Tika tika;
    private final DocumentRepository documentRepository;
    private final ChunkService chunkService;
    private final EmbeddingService embeddingService;

    @Transactional
    public Document uploadDocument(String knowledgeId, MultipartFile file) {
        String documentId = IdGenerator.generate("doc");
        String minioPath = buildMinioPath(knowledgeId, documentId, file.getOriginalFilename());

        // 1. 上传 MinIO
        try {
            minioClient.putObject(PutObjectArgs.builder()
                    .bucket("knowledge-docs")
                    .object(minioPath)
                    .stream(file.getInputStream(), file.getSize(), -1)
                    .contentType(file.getContentType())
                    .build());
        } catch (Exception e) {
            throw new SystemException("文件上传失败", e);
        }

        // 2. 写入 t_document 表
        Document doc = Document.builder()
                .documentId(documentId)
                .tenantId(TenantContext.getCurrentTenantId())
                .knowledgeId(knowledgeId)
                .filename(file.getOriginalFilename())
                .fileType(getFileType(file.getOriginalFilename()))
                .fileSize(file.getSize())
                .minioPath(minioPath)
                .status("UPLOADING")
                .uploadedBy(TenantContext.getCurrentUserId())
                .build();
        documentRepository.save(doc);

        // 3. 异步处理
        asyncProcess(documentId);

        return doc;
    }

    @Async
    public void asyncProcess(String documentId) {
        try {
            // 解析
            updateStatus(documentId, "PARSING");
            String text = parseDocument(documentId);

            // 切分
            updateStatus(documentId, "CHUNKING");
            List<Chunk> chunks = chunkService.split(text, 512, 50);

            // 向量化
            updateStatus(documentId, "EMBEDDING");
            embeddingService.embedAndStore(documentId, chunks);

            // 完成
            updateStatus(documentId, "READY");
            documentRepository.updateChunkCount(documentId, chunks.size());
            knowledgeBaseRepository.incrementDocumentCount(knowledgeId);
        } catch (Exception e) {
            log.error("文档处理失败: {}", documentId, e);
            updateStatus(documentId, "FAILED");
            documentRepository.updateErrorMessage(documentId, e.getMessage());
        }
    }

    private String parseDocument(String documentId) throws Exception {
        Document doc = documentRepository.findByDocumentId(documentId);
        InputStream is = minioClient.getObject(GetObjectArgs.builder()
                .bucket("knowledge-docs").object(doc.getMinioPath()).build());
        return tika.parseToString(is);
    }
}
```

### 3. 文件类型校验

| 支持格式 | MIME Type |
|----------|-----------|
| PDF | `application/pdf` |
| DOCX | `application/vnd.openxmlformats-officedocument.wordprocessingml.t_document` |
| TXT | `text/plain` |
| Markdown | `text/markdown` |
| HTML | `text/html` |
| CSV | `text/csv` |

单个文件上限：**50MB**，单知识库文档上限：**1000 个**
