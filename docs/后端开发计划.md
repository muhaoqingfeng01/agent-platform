# Agent 平台后端开发计划（按优先级）

> 基于《企业级 Agent 平台技术方案》分析，按依赖关系和业务价值排列优先级。
> 本项目仅包含后端，无前端页面，所有接口通过 Swagger 调试。
> 前端交互层见 `docs/P5-交互端/`。

---

## 优先级总览

```
P0 (基础底座) ──→ P1 (核心能力) ──→ P2 (增强能力) ──→ P3 (安全治理) ──→ P4 (观测优化)
     ↑                  ↑                ↑                ↑                ↑
 统一网关+多租户    意图+提示词+DAG    RAG+MCP        安全+审批       观测+评估
  必须先做          对话的核心         让Agent         生产环境         持续运营
                   没有P0做不了       真正有用         必备能力         必备能力
```

| 优先级 | 内容 | 依赖 | 预计工时 | 核心价值 |
|:--:|------|------|:--:|------|
| **P0** | 统一网关 + 多租户 | 无 | 5天 | 安全底座 + 多租户骨架，没这个其他都跑不了 |
| **P1** | 意图识别 + 提示词 + 任务规划 | P0 | 8天 | Agent 核心对话能力，产品的最小可用版本 |
| **P2** | RAG 知识库 + MCP 工具 | P1 | 6天 | 知识检索 + 工具调用，让 Agent 真正能干活的增强 |
| **P3** | 安全围栏 + 人机协同 | P2 | 3天 | 安全合规 + 高风险管控，生产环境必备 |
| **P4** | 全链路观测 + 效果评估 | P3 | 4天 | 可观测 + 持续优化，数据驱动迭代 |
| **P5** | 交互端构建（前端） | 独立 | — | Web聊天/审批/反馈/IM，与后端并行开发 |

---

## P0 — 基础底座（必须先做）

### P0-T8 多租户 + RBAC 权限

**目标**：所有业务数据按租户隔离，用户通过角色权限访问资源。

| 序号 | 功能点 | 实现内容 | 涉及文件 |
|:--:|--------|----------|----------|
| 1 | 租户 CRUD | `TenantController` 提供租户创建/查询/启停/删除接口 | `interfaces/rest/admin/TenantController.java` |
| 2 | 租户拦截器 | `TenantInterceptor` 从 Sa-Token 提取 tenant_id，注入 `TenantContext` | `infrastructure/security/filter/TenantInterceptor.java` |
| 3 | 用户 CRUD | `UserController` 用户注册、查询、启停，密码 BCrypt 加密 | `interfaces/rest/admin/UserController.java` |
| 4 | 认证接口 | `AuthController` 登录/登出/刷新 Token | `interfaces/rest/admin/AuthController.java` |
| 5 | RBAC 权限 | `RoleController` + `PermissionController` 角色/权限 CRUD + 关联 | `interfaces/rest/admin/RoleController.java` |
| 6 | MyBatis 自动注入 | SQL 拦截器自动给查询加 `tenant_id = ?` 条件 | `infrastructure/config/mybatis/TenantSqlInterceptor.java` |
| 7 | DTO 转换 | request/response DTO + MapStruct Converter | `interfaces/dto/` + `infrastructure/persistence/converter/` |
| 8 | 全局异常处理 | `GlobalExceptionHandler` 统一错误响应格式（Result<T>） | `common/exception/GlobalExceptionHandler.java` |
| 9 | 统一响应体 | `Result<T>` 封装 code/t_message/data/timestamp | `common/result/Result.java` |

**涉及的数据库表**：`t_tenant`, `t_user`, `t_role`, `t_permission`, `t_user_role`, `t_role_permission`

**Swagger 标签**：`租户管理`

**验证方式**：Swagger 上创建租户 → 创建用户 → 登录获取 Token → 用 Token 调其他接口

---

### P0-T2 统一网关与安全流控

**目标**：Sa-Token 鉴权 + Sentinel 限流熔断，确保 API 安全可控。

| 序号 | 功能点 | 实现内容 | 涉及文件 |
|:--:|--------|----------|----------|
| 1 | Sa-Token 工具类 | `StpUtil` 生成/解析/校验 Token，支持过期刷新 | `infrastructure/security/StpUtil.java` |
| 2 | Sa-Token 认证过滤器 | `SaTokenPathFilter` 从 Header 提取 Token，注入 StpUtil.getSession() | `infrastructure/security/filter/SaTokenPathFilter.java` |
| 3 | Sa-Token 配置 | `SecurityConfig` 配置白名单、鉴权规则、CORS | `infrastructure/config/security/SecurityConfig.java` |
| 4 | 限流注解 | `@RateLimit` 自定义注解 + AOP 切面，基于 Sentinel 实现 | `common/aspect/RateLimitAspect.java` |
| 5 | 熔断降级 | `SentinelConfig` 配置熔断规则，错误率>50% 熔断 30 秒 | `infrastructure/config/security/SentinelConfig.java` |
| 6 | 统一 CORS | Gateway/Filter 层处理跨域 | `infrastructure/config/web/CorsConfig.java` |

**Swagger 标签**：认证相关接口在 `租户管理` 下

**验证方式**：不带 Token 调任意 API → 401，带有效 Token → 200，频繁调用限流接口 → 429

---

> **P0 完成标志**：可以用 Swagger 创建租户和用户，登录拿到 Token，用 Token 调需要认证的接口。限流规则生效。

## P1 — 核心能力（最小可用版本）

### P1-T3 意图识别与对话管理

**目标**：实现 Agent 对话核心流程——接收消息 → 识别意图 → 管理状态 → 返回回复。

| 序号 | 功能点 | 实现内容 | 涉及文件 |
|:--:|--------|----------|----------|
| 1 | 会话管理 | `ConversationController` 创建会话、查询历史、关闭/归档 | `interfaces/rest/t_conversation/ConversationController.java` |
| 2 | 消息收发 | `MessageController` 发送消息、查询历史消息列表 | `interfaces/rest/t_conversation/MessageController.java` |
| 3 | 流式响应(SSE) | `ConversationController.streamChat()` 返回 `text/event-stream`，逐 token 推送 | `interfaces/rest/t_conversation/ConversationController.java` |
| 4 | 意图识别引擎 | `IntentRecognitionService` 规则优先（正则/关键词匹配）+ LLM 兜底 | `application/orchestration/IntentRecognitionService.java` |
| 5 | 意图 CRUD | `IntentController` 意图定义的管理接口 | `interfaces/rest/admin/IntentController.java` |
| 6 | 对话状态机 | `ConversationStateMachine` 基于 Spring StateMachine 的状态流转 | `domain/t_conversation/service/ConversationStateMachine.java` |
| 7 | 短期记忆 | `SessionMemoryService` Redis 存取最近 N 轮对话，TTL 30 分钟 | `infrastructure/repository/memory/RedisSessionMemoryRepository.java` |
| 8 | 长期记忆 | `LongTermMemoryService` 提取用户事实异步写入 MySQL | `application/memory/LongTermMemoryService.java` |
| 9 | WebSocket 推送 | `ConversationWebSocketHandler` 推送消息、审批卡片、执行进度 | `interfaces/websocket/ConversationWebSocketHandler.java` |

**涉及的数据库表**：`t_conversation`, `t_message`, `t_intent`, `t_long_term_memory`

**Swagger 标签**：`对话管理`、`意图管理`

**验证方式**：Swagger 创建会话 → 发消息 → 看到 SSE 流式回复 → 查询历史消息和长期记忆提取结果

### P1-T4 提示词管理与版本控制

**目标**：提示词模板在线管理，版本发布回滚，运行时动态渲染。

| 序号 | 功能点 | 实现内容 | 涉及文件 |
|:--:|--------|----------|----------|
| 1 | 模板 CRUD | `PromptTemplateController` 创建/编辑/查询/删除模板 | `interfaces/rest/prompt/PromptTemplateController.java` |
| 2 | 变量预览 | `POST /prompts/{id}/preview` 输入变量值，返回渲染后的完整提示词 | 同上 |
| 3 | 版本发布 | 发布时版本号 +1，旧版本写入 `t_prompt_template_version` 保留 | `application/prompt/PromptTemplateService.java` |
| 4 | 版本回滚 | `POST /prompts/{id}/rollback?version=N` 回滚到指定历史版本 | 同上 |
| 5 | 版本历史查询 | `GET /prompts/{id}/versions` 列出所有历史版本 | `interfaces/rest/prompt/PromptTemplateController.java` |
| 6 | 运行时渲染 | `PromptService.render(name, version, context)` 结合上下文变量渲染最终 Prompt | `application/prompt/PromptRenderService.java` |
| 7 | Redis 缓存 | 热点模板缓存到 Redis，发布/回滚时自动失效 | `infrastructure/repository/prompt/CachedPromptRepository.java` |

**涉及的数据库表**：`t_prompt_template`, `t_prompt_template_version`

**Swagger 标签**：`提示词管理`

**验证方式**：创建模板 → 版本预览 → 发布 → 再编辑 → 发布(v2) → 回滚到 v1 → 验证渲染结果

---

### P1-T5 任务规划与执行引擎

**目标**：对复杂意图，LLM 规划 DAG 任务序列，多步骤并行执行，失败重试。

| 序号 | 功能点 | 实现内容 | 涉及文件 |
|:--:|--------|----------|----------|
| 1 | 任务规划 Prompt | 专用提示词要求 LLM 返回 JSON 格式任务序列和依赖关系 | `application/orchestration/TaskPlanningService.java` |
| 2 | DAG 解析器 | `DagParser` 解析 LLM 返回的 JSON，构建 DAG 图，检测循环依赖 | `application/orchestration/DagParser.java` |
| 3 | DAG 执行器 | `DagExecutor` 无依赖节点并行执行，使用 `CompletableFuture` 编排 | `application/orchestration/DagExecutor.java` |
| 4 | ActionHandler 注册 | `ActionHandlerRegistry` 自动扫描所有 `ActionHandler` 实现类 | `application/orchestration/ActionHandlerRegistry.java` |
| 5 | 失败重试 | 节点失败自动重试（最多 3 次），指数退避 | `application/orchestration/RetryPolicy.java` |
| 6 | 执行追踪 | 实时同步执行状态至 Redis + MySQL，WebSocket 推送进度 | `infrastructure/repository/agent/TaskExecutionRepository.java` |
| 7 | 执行历史查询 | `TaskExecutionController` 查询历史执行记录和详情 | `interfaces/rest/agent/TaskExecutionController.java` |

**涉及的数据库表**：`t_task_execution`, `t_task_step_execution`

**Swagger 标签**：`任务引擎`

**验证方式**：发复杂消息触发任务规划 → 看 DAG 计划 → 查执行进度 → 验证并行执行和重试

---

> **P1 完成标志**：完整的 Agent 对话闭环——接收消息 → 识别意图 → 加载提示词 → 规划任务 → 执行工具链 → 流式返回结果。Swagger 上可完整体验。

## P2 — 增强能力

### P2-T6 RAG 知识库引擎

**目标**：文档上传 → 解析切分 → 向量化 → 存入 Milvus → 混合检索 → 注入上下文。

| 序号 | 功能点 | 实现内容 | 涉及文件 |
|:--:|--------|----------|----------|
| 1 | 知识库 CRUD | `KnowledgeBaseController` 创建/查询/删除知识库 | `interfaces/rest/knowledge/KnowledgeBaseController.java` |
| 2 | 文档上传 | `POST /knowledge/{id}/documents` multipart 上传，存 MinIO | 同上 |
| 3 | 文档解析切分 | `DocumentPipelineService` 异步 Tika 解析 → 按语义切 chunk(512t/50t 重叠) | `application/knowledge/DocumentPipelineService.java` |
| 4 | 向量化写入 | `EmbeddingService` 调 Embedding 模型 → 写入 Milvus Collection `kb_{tenantId}` | `infrastructure/repository/knowledge/MilvusVectorRepository.java` |

**Swagger 标签**：`知识库`

## P3 — 安全治理

### P3-T10 安全围栏

**目标**：输入过滤（防注入/越狱）→ 输出脱敏（PII 识别替换）→ 责任链拦截。

| 序号 | 功能点 | 实现内容 | 涉及文件 |
|:--:|--------|----------|----------|
| 1 | 输入过滤器链 | `InputFilterChain` 责任链：注入检测 → 越狱检测 → 敏感词匹配 → 长度限制 | `infrastructure/guardrails/input/InputFilterChain.java` |
| 2 | 敏感词管理 | `SensitiveWordController` 敏感词 CRUD + CSV 批量导入 | `interfaces/rest/admin/SensitiveWordController.java` |
| 3 | PII 脱敏 | `PiiDesensitizer` 正则识别身份证/手机/邮箱，替换为 `***` | `infrastructure/guardrails/output/PiiDesensitizer.java` |
| 4 | 输出过滤器 | `OutputFilterChain` 责任链：PII 检测 → 敏感词检测 → 长度截断 | `infrastructure/guardrails/output/OutputFilterChain.java` |
| 5 | 安全事件记录 | 每次拦截写入 `t_security_event` 表，高危触发告警 | `application/notification/SecurityEventService.java` |

**涉及的数据库表**：`t_sensitive_word`, `t_security_event`

**Swagger 标签**：`安全审计`

**验证方式**：添加敏感词 → 发送含敏感词的消息 → 验证被拦截 → 查安全事件记录

## P4 — 可观测与优化

|:--:|--------|----------|----------|

| 3 | 审计日志切面 | `@Auditable` 注解 + AOP 切面，自动记录关键操作到 `t_audit_log` | `common/aspect/AuditLogAspect.java` |
| 4 | Prometheus 指标 | Micrometer 暴露 QPS/延迟/错误率，自定义业务指标 | `infrastructure/config/web/MetricsConfig.java` |

**涉及的数据库表**：`t_audit_log`

**Swagger 标签**：`健康检查`（actuator 端点）、`安全审计`

**验证方式**：调任意 API → 响应 Header 有 traceId → 查审计日志有记录 → `/actuator/prometheus` 有指标

## Swagger 配置说明

Swagger UI 已在项目中集成，配置如下：

- **依赖**：`springdoc-openapi-starter-webmvc-ui` 2.7.0（已在 `interfaces/pom.xml`）
- **配置类**：`infrastructure/config/web/OpenApiConfig.java`
- **YAML 配置**：`application.yml` 中 `springdoc.*` 段
- **访问地址**：启动后访问 `http://localhost:8080/swagger-ui.html`
- **OpenAPI JSON**：`http://localhost:8080/v3/api-docs`
- **认证方式**：Swagger 页面右上角 "Authorize" 按钮输入 Sa-Token
