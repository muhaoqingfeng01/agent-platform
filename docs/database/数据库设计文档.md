## 1. 多租户与权限（T2/T8）

### ✅ t_tenant（租户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 自增主键 |
| tenant_id | BIGINT UNIQUE | 租户唯一标识 |
| name | VARCHAR(128) | 租户名称 |
| status | VARCHAR(32) | ACTIVE / SUSPENDED / DELETED |
| tier | VARCHAR(32) | STANDARD / PREMIUM / ENTERPRISE |
| config_json | TEXT | 租户级配置 JSON（限流阈值、模型偏好等） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |
| deleted | TINYINT | 逻辑删除 |

### ✅ t_user（用户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 自增主键 |
| tenant_id | BIGINT | 所属租户 |
| user_id | VARCHAR(64) UNIQUE | 用户唯一标识 |
| username | VARCHAR(64) | 用户名 |
| password_hash | VARCHAR(256) | 密码哈希 |
| email | VARCHAR(128) | 邮箱 |
| phone | VARCHAR(32) | 手机号 |
| status | VARCHAR(32) | ACTIVE / DISABLED |
| created_at / updated_at / deleted | - | 通用字段 |

### ✅ t_role（角色表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 自增主键 |

| role_name | VARCHAR(128) | 角色名称 |

### ✅ t_permission（权限表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 自增主键 |
| permission_code | VARCHAR(128) UNIQUE | 权限编码 |
| resource | VARCHAR(128) | 资源路径 |

## 4. Agent 配置与任务执行（T5）

### ✅ t_agent_config（Agent 配置表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 自增主键 |
| agent_id | VARCHAR(64) UNIQUE | Agent 唯一标识 |
| tenant_id | BIGINT | 所属租户 |
| name | VARCHAR(128) | Agent 名称 |
| description | TEXT | 描述 |
| system_prompt | TEXT | 系统提示词 |
| model_config | JSON | `{"provider":"dashscope","model":"qwen-max","temperature":0.7}` |
| tool_ids | JSON | 关联工具 ID 列表 |
| knowledge_ids | JSON | 关联知识库 ID 列表 |
| status | VARCHAR(32) | DRAFT / ACTIVE / PAUSED / ARCHIVED |
| version | INT | 配置版本号 |

### 🆕 t_task_execution（任务执行记录表）

> 对应 T5：DAG 执行器监控节点状态，记录每次复杂任务的执行过程。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 自增主键 |
| tenant_id | BIGINT | 所属租户 |
| execution_id | VARCHAR(64) UNIQUE | 执行唯一标识 |
| conversation_id | VARCHAR(64) | 关联会话 |
| agent_id | VARCHAR(64) | 关联 Agent |
| plan_json | JSON | DAG 任务计划 `[{"id":"1","action":"xxx","dep":[]}]` |
| status | VARCHAR(32) | PENDING / RUNNING / COMPLETED / FAILED / CANCELLED |
| total_steps | INT | 总步骤数 |
| completed_steps | INT | 已完成步骤数 |
| failed_step_id | VARCHAR(32) | 失败步骤 ID |
| error_message | TEXT | 错误信息 |
| started_at | DATETIME | 开始时间 |
| finished_at | DATETIME | 完成时间 |

### 🆕 t_task_step_execution（任务步骤执行详情表）

> 对应 T5：每个 DAG 节点的执行追踪，支持失败重试。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 自增主键 |
| execution_id | VARCHAR(64) | 关联执行记录 |
| step_id | VARCHAR(32) | 步骤 ID（对应 plan_json 中的 id） |
| action | VARCHAR(128) | 动作类型（如 retrieve_order、send_email） |
| handler_class | VARCHAR(256) | ActionHandler 实现类 |
| input_json | JSON | 输入参数 |
| output_json | JSON | 输出结果 |
| status | VARCHAR(32) | PENDING / RUNNING / SUCCESS / FAILED / SKIPPED |
| retry_count | INT DEFAULT 0 | 已重试次数 |
| max_retries | INT DEFAULT 3 | 最大重试次数 |
| duration_ms | BIGINT | 执行耗时（毫秒） |
| started_at | DATETIME | 开始时间 |
| finished_at | DATETIME | 完成时间 |

**索引**: `idx_step_exec (execution_id)`, `idx_step_status (status)`

> 对应 T6：文档入库 → MinIO 存储 → Tika 解析 → 切分。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 自增主键 |
| tenant_id | BIGINT | 所属租户 |
| knowledge_id | VARCHAR(64) | 所属知识库 |

| content_hash | VARCHAR(128) | 文件内容 SHA256 |
| chunk_count | INT DEFAULT 0 | 切分块数 |
| status | VARCHAR(32) | UPLOADING / PARSING / CHUNKING / EMBEDDING / READY / FAILED |

| uploaded_at | DATETIME | 上传时间 |
| created_at / updated_at / deleted | - | 通用字段 |

### 🆕 t_document_chunk（文档切片表）

> 对应 T6：每个 chunk 的元数据存储在 MySQL，向量存储在 Milvus。

| created_at | DATETIME | 创建时间 |

### 🆕 t_knowledge_hit_record（知识命中记录表）

| conversation_id | VARCHAR(64) | 关联会话 |

## 7. 安全围栏与审计（T10）

### 🆕 t_sensitive_word（敏感词库表）

> 对应 T10：输入过滤——基于规则和敏感词库过滤危险输入。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 自增主键 |
| tenant_id | BIGINT | 所属租户（NULL=全局） |
| word | VARCHAR(256) | 敏感词/正则表达式 |
| match_type | VARCHAR(32) | EXACT / REGEX / SEMANTIC |
| category | VARCHAR(64) | INJECTION / JAILBREAK / PII / CUSTOM |
| severity | VARCHAR(32) | LOW / MEDIUM / HIGH / BLOCK |
| action | VARCHAR(32) | LOG / WARN / BLOCK |
| status | VARCHAR(32) | ACTIVE / DISABLED |
| created_at / updated_at | - | 通用字段 |

### 🆕 t_security_event（安全事件表）

> 对应 T10：安全拦截事件审计，支持追溯和合规审查。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 自增主键 |
| tenant_id | BIGINT | 所属租户 |
| event_type | VARCHAR(64) | INPUT_FILTER / OUTPUT_DESENSITIZE / INJECTION_DETECTED |
| rule_id | BIGINT | 触发的规则 ID |
| conversation_id | VARCHAR(64) | 关联会话 |
| message_id | VARCHAR(64) | 关联消息 |
| original_content | TEXT | 原始内容 |
| processed_content | TEXT | 处理后内容（脱敏后） |
| matched_pattern | VARCHAR(512) | 匹配到的模式 |
| action_taken | VARCHAR(32) | LOG / WARN / BLOCK |
| operator | VARCHAR(64) | 操作人（SYSTEM / 具体用户） |
| created_at | DATETIME | 事件时间 |

**索引**: `idx_security_event_type (event_type)`, `idx_security_event_tenant (tenant_id, created_at)`

### 🆕 t_audit_log（综合审计日志表）

> 对应 T9/T10：所有用户输入、模型输出、工具参数写入审计存储，保留 90 天以上。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 自增主键 |
| tenant_id | BIGINT | 所属租户 |
| trace_id | VARCHAR(64) | 全链路追踪 ID |
| conversation_id | VARCHAR(64) | 关联会话 |
| actor_type | VARCHAR(32) | USER / ASSISTANT / TOOL / SYSTEM |
| actor_id | VARCHAR(64) | 操作者标识 |
| action | VARCHAR(128) | 动作类型 |
| resource_type | VARCHAR(64) | 资源类型 |
| resource_id | VARCHAR(128) | 资源标识 |
| request_json | JSON | 请求内容 |
| response_json | JSON | 响应内容 |
| duration_ms | BIGINT | 耗时 |
| status | VARCHAR(32) | SUCCESS / FAILED |
| ip_address | VARCHAR(64) | 客户端 IP |
| user_agent | VARCHAR(512) | User-Agent |
| created_at | DATETIME | 记录时间 |

**索引**: `idx_audit_trace (trace_id)`, `idx_audit_tenant_time (tenant_id, created_at)`, `idx_audit_conv (conversation_id)`

> ⚠️ 注意：对于超大规模部署，建议将 `t_audit_log` 和 `t_security_event` 数据异步写入 Elasticsearch 而非 MySQL，以支撑高性能检索和长期保留。MySQL 仅保留近期热数据（如 30 天）。
