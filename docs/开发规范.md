# 一、编程规约

## 1.1 命名风格

### 🔴【强制】通用命名

| 规则 | 正例 | 反例 |
|------|------|------|
| 命名不能以 `_` 或 `$` 开始/结束 | `name`, `Object` | `_name`, `name_`, `$name` |
| 严禁拼音与英文混合，禁止纯中文 | `alibaba`, `taobao` | `DaZhePromotion [打折]` |
| 代码中禁止使用歧视性词语 | `blockList`, `allowList` | `blackList`, `whiteList`, `slave` |

### 🔴【强制】类/方法/变量/常量命名

| 类型 | 规则 | 正例 | 反例 |
|------|------|------|------|
| 类名 | **UpperCamelCase** | `UserController`, `TenantPO` | `userController` |
| 方法/参数/变量 | **lowerCamelCase** | `localValue`, `getUserById()` | `LocalValue` |
| 常量 | **全大写 + 下划线** | `MAX_RETRY_COUNT` | `MaxRetryCount` |
| 抽象类 | `Abstract` 或 `Base` 开头 | `AbstractUserService` | — |
| 异常类 | `Exception` 结尾 | `ResourceNotFoundException` | — |
| 测试类 | `Test` 结尾 | `UserServiceTest` | — |
| 数组定义 | 类型与 `[]` 紧挨 | `int[] arrayDemo` | `int arrayDemo[]` |

### 🔴【强制】POJO 布尔变量不加 is 前缀

```java
// ✅ 正例
Boolean deleted;    // 生成的 getter: getDeleted()

// ❌ 反例
Boolean isDeleted;  // RPC 框架反向解析时误认为属性名是 deleted，导致序列化异常
```

### 🔴【强制】Service/DAO 层方法命名

| 操作 | 前缀 | 示例 |
|------|------|------|
| 获取单个 | `get` | `getUser(Long id)` |
| 获取多个 | `list` | `listUsers(int page, int size)` |
| 获取统计 | `count` | `countActiveUsers()` |
| 插入 | `save` / `insert` | `saveUser(User user)` |
| 删除 | `remove` / `delete` | `removeUser(Long id)` |
| 修改 | `update` / `modify` | `updateUser(User user)` |
| 仓储查单个 | `find` | `findById(Long id)` |
| 仓储查多个 | `find` | `findByTenantId(String tid)` |

### 🔴【强制】项目领域模型命名

| 类型 | 命名 | 包路径 |
|------|------|--------|
| 领域实体 | `{Entity}` | `domain.tenant` / `domain.security` |
| 仓储接口 | `{Entity}Repository` | `domain.tenant` |
| 领域服务接口 | `{Entity}Service` | `domain.security` |
| 持久化对象 | `{Entity}PO` | `infrastructure.persistence.po` |
| MyBatis Mapper | `{Entity}Mapper` | `infrastructure.persistence.mapper` |
| 仓储实现(MyBatis) | `{Entity}RepositoryImpl` | `infrastructure.persistence.impl` |
| 仓储实现(Stub) | `{Entity}RepositoryStub` | `infrastructure.persistence.stub` |
| 领域服务实现 | `{Entity}ServiceImpl` | `infrastructure.security` |
| Controller | `{Entity}Controller` | `interfaces.rest` |
| 请求体 | `{Action}{Entity}Request` | `interfaces.dto` |
| 响应体 | `{Action}{Entity}Response` | `interfaces.dto` |

### 🟡【推荐】接口与实现类命名

```java
// ✅ Service/Repository 以接口形式暴露，实现类加 Impl 后缀
UserRepository userRepository;  // 接口
UserRepositoryImpl implements UserRepository  // 实现

// ✅ 形容能力的接口使用 -able 结尾
public interface Translatable { ... }
```

### 🟡【推荐】类型名词放词尾

| ✅ 正例 | ❌ 反例 |
|---------|---------|
| `workQueue` | `QueueOfWork` |
| `nameList` | `listName` |
| `startTime` | `startedAt` |

## 1.2 常量定义

### 🔴【强制】禁止魔法值直接出现在代码中

```java
// ❌ 反例：魔法值硬编码
String key = "Id#taobao_" + tradeId;
cache.put(key, value);
// 同事 B 使用时少了下划线 → key = "Id#taobao" + tradeId → 缓存 key 不一致 → 线上故障

// ✅ 正例：提取为常量
private static final String CACHE_KEY_PREFIX = "Id#taobao_";
String key = CACHE_KEY_PREFIX + tradeId;
```

### 🔴【强制】long/Long 字面量使用大写 L

```java
// ❌ 反例
Long a = 2l;  // 是数字 21 还是 Long 型 2？易混淆

// ✅ 正例
Long a = 2L;
```

### 🟡【推荐】按功能归类常量，禁止大而全的常量类

```java
// ✅ 正例
public final class CacheConsts {       // 缓存常量
    public static final long DEFAULT_TTL_SECONDS = 3600;
}
public final class SecurityConsts {    // 安全常量
    public static final int BCRYPT_STRENGTH = 12;
}

// ❌ 反例
public final class Constants {  // 所有常量堆在一起
    public static final int CACHE_TTL = 3600;
    public static final int BCRYPT_STRENGTH = 12;
    public static final String DEFAULT_TIER = "STANDARD";
}
```

### 🟡【推荐】变量值在固定范围内变化时使用枚举

```java
// ✅ 正例
public enum TierEnum {
    STANDARD(1, "标准版"),
    PREMIUM(2, "高级版"),
    ENTERPRISE(3, "企业版");
}
```

## 1.3 OOP 规约

### 🔴【强制】重写 equals 必须重写 hashCode

```java
// ✅ 正例：User 实体同时重写 equals 和 hashCode
@Override
public boolean equals(Object o) {
    if (this == o) return true;
    if (!(o instanceof User user)) return false;
    return Objects.equals(userId, user.userId);
}

@Override
public int hashCode() {
    return Objects.hash(userId);
}
```

### 🔴【强制】Object.equals 避免 NPE

```java
// ✅ 正例：常量或确定非 null 的对象放前面
"expectedValue".equals(actualValue);
Objects.equals(a, b);  // JDK7+ 推荐
```

### 🔴【强制】包装类比较使用 equals，不用 ==

```java
// ❌ 反例
Integer a = 128;
Integer b = 128;
boolean eq = a == b;  // false! -128~127 之外的对象 == 比较的是引用

// ✅ 正例
boolean eq = a.equals(b);
```

### 🔴【强制】POJO 类必须覆写 toString

**说明:** 打印日志、调试、异常排查时必须能清晰看到对象内容。使用 Lombok `@Data` 或 `@ToString`。

### 🔴【强制】禁止在父类构造方法中调用子类可覆写的方法

### 🟡【推荐】类成员与方法访问控制从严

```java
// 优先级: private → package-private(default) → protected → public
// 能 private 则不 public，能包级可见则不对外暴露
```

## 1.4 集合处理

### 🔴【强制】判断集合为空使用 isEmpty() 而非 size() == 0

```java
// ✅ 正例
if (map.isEmpty()) { ... }

// ❌ 反例
if (map.size() == 0) { ... }
```

### 🔴【强制】Collectors.toMap() 必须指定 mergeFunction

```java
// ❌ 反例：key 重复时抛 IllegalStateException
Map<Integer, String> map = list.stream()
    .collect(Collectors.toMap(Item::getId, Item::getName));

// ✅ 正例：指定冲突处理策略
Map<Integer, String> map = list.stream()
    .collect(Collectors.toMap(Item::getId, Item::getName, (v1, v2) -> v2));  // 保留后者
```

### 🔴【强制】toMap() 时 value 不能为 null

```java
// ❌ 反例：value 为 null 时 toMap() 抛 NullPointerException
list.stream().collect(Collectors.toMap(k -> k, v -> nullValue));  // NPE!
```

### 🔴【强制】Arrays.asList() 返回的集合不可增删

```java
// ❌ 反例
List<String> list = Arrays.asList("a", "b");
list.add("c");  // UnsupportedOperationException!
```

### 🔴【强制】ArrayList.subList() 不可强转为 ArrayList

```java
// ❌ 反例
ArrayList<String> sub = (ArrayList<String>) list.subList(0, 2);
// ClassCastException: RandomAccessSubList cannot be cast to ArrayList
```

### 🔴【强制】foreach 循环中不可进行 remove/add 操作

```java
// ❌ 反例
for (String item : list) {
    if (condition) list.remove(item);  // ConcurrentModificationException!
}

// ✅ 正例：使用 Iterator
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String item = it.next();
    if (condition) it.remove();
}
```

### 🔴【强制】集合转数组使用 toArray(T[] array)

```java
// ✅ 正例（最佳性能）
String[] array = list.toArray(new String[0]);

// ❌ 反例
String[] array = (String[]) list.toArray();  // ClassCastException!
```

### 🔴【强制】addAll() 必须对输入集合做 NPE 判断

### 🟡【推荐】集合初始化指定初始容量

```java
// initialCapacity = (预期元素数 / 0.75) + 1
HashMap<String, Object> map = new HashMap<>(16);
```

### 🟡【推荐】使用 entrySet 遍历 Map

```java
// ✅ 正例：entrySet 遍历一次
for (Map.Entry<String, Object> entry : map.entrySet()) { ... }

// JDK8 更推荐
map.forEach((k, v) -> { ... });

// ❌ 反例：keySet 多一次 get 调用
for (String key : map.keySet()) { map.get(key); }
```

### 🔴【强制】Map key/value null 存储差异

| 集合类 | Key null | Value null | 线程安全 |
|--------|:--:|:--:|:--:|
| HashMap | ✅ | ✅ | ❌ |
| ConcurrentHashMap | ❌ | ❌ | ✅ |
| TreeMap | ❌ | ✅ | ❌ |

### 🔴【强制】使用 Comparator 必须满足自反性、对称性、传递性

```java
// ❌ 反例：相等时未返回 0
(o1, o2) -> o1.getId() > o2.getId() ? 1 : -1

// ✅ 正例
(o1, o2) -> Long.compare(o1.getId(), o2.getId())
```

## 1.5 并发处理

### 🔴【强制】线程池必须用 ThreadPoolExecutor 创建，禁用 Executors

```java
// ❌ 反例
ExecutorService pool = Executors.newFixedThreadPool(10);
// FixedThreadPool 队列长度 Integer.MAX_VALUE → 可能 OOM
// CachedThreadPool 最大线程数 Integer.MAX_VALUE → 可能 OOM

// ✅ 正例
ThreadPoolExecutor pool = new ThreadPoolExecutor(
    10, 20, 60L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(1000),
    new ThreadFactoryBuilder().setNameFormat("agent-pool-%d").build(),
    new ThreadPoolExecutor.CallerRunsPolicy()
);
```

### 🔴【强制】创建线程必须指定有意义的线程名称

```java
// ✅ 正例
ThreadFactory factory = new ThreadFactoryBuilder()
    .setNameFormat("agent-worker-%d")
    .build();
```

### 🔴【强制】SimpleDateFormat 不是线程安全类，禁作 static 变量

```java
// ❌ 反例
private static final SimpleDateFormat SDF = new SimpleDateFormat("yyyy-MM-dd");

// ✅ 正例：JDK8 使用 DateTimeFormatter（线程安全）
private static final DateTimeFormatter DTF = DateTimeFormatter.ofPattern("yyyy-MM-dd");
```

### 🔴【强制】ThreadLocal 必须在使用后回收

```java
// ✅ 正例
TenantContext.setTenantId(tenantId);
try {
    doBusiness();
} finally {
    TenantContext.clear();  // 必须清理！否则线程池场景下内存泄漏 + 数据串扰
}
```

### 🔴【强制】Lock 必须在 try 代码块之外加锁

```java
// ✅ 正例
lock.lock();          // 加锁在 try 外
try {
    doSomething();
} finally {
    lock.unlock();    // 解锁在 finally 中
}

// ❌ 反例
try {
    lock.lock();      // 如果这里之前抛异常，finally 会 unlock 未加锁的对象
    doSomething();
} finally {
    lock.unlock();
}
```

### 🔴【强制】多线程定时任务使用 ScheduledExecutorService 而非 Timer

**说明:** Timer 中任一任务抛未捕获异常会导致所有定时任务终止。

### 🔴【强制】获取单例对象必须保证线程安全

### 🔴【强制】并发修改同一记录必须加锁

**手段:** 应用层锁 / 缓存分布式锁 / 数据库乐观锁（version 字段）

### 🟡【推荐】资金相关使用悲观锁，非资金场景可用乐观锁

### 🔴【强制】switch 块必须有 default 分支（即使为空）

### 🔴【强制】if/for/while 后必须跟 {}，即使只有一行

### 🔴【强制】禁止出现死循环 / 无限递归

```java
// ✅ 正例
StringBuilder sb = new StringBuilder();
for (String item : list) { sb.append(item).append(","); }
// ❌ 反例
String result = "";
for (String item : list) { result += item + ","; }  // 每次循环创建新 String 对象
```

// ✅ 正例：列表查询无结果返回 []
return Result.ok(Collections.emptyList());

# 二、异常与日志

## 2.1 异常处理

### 🔴【强制】runtimeException 不应被 catch，应用 NPE 和下标越界应在编码阶段避免

### 🔴【强制】异常不可用于流程控制

```java
// ❌ 反例：用异常控制业务分支
try { obj = list.get(0); } catch (IndexOutOfBoundsException e) { return null; }

// ✅ 正例
if (list.isEmpty()) return null;
obj = list.get(0);
```

### 🔴【强制】try-catch 大段代码：catch 后应记录完整异常栈

```java
// ✅ 正例
log.error("处理订单失败: orderId={}", orderId, e);  // 最后一个参数传入异常对象

// ❌ 反例
log.error("处理订单失败: {}", e.getMessage());  // 丢失异常栈信息
```

### 🔴【强制】禁止吞异常

```java
// ❌ 反例：空 catch 块
try { ... } catch (Exception e) { }  // 什么都不做

// ✅ 正例：至少打印日志
try { ... } catch (Exception e) {
    log.warn("忽略非关键异常", e);
}
```

### 🔴【强制】Controller 层不 try-catch → 全部由 GlobalExceptionHandler 处理

### 🔴【强制】Service 层抛语义化业务异常

```java
throw new ResourceNotFoundException("用户", userId);
throw new BusinessException(400, "不支持的操作类型: " + type);
```

### 🔴【强制】禁止 `e.printStackTrace()` → 统一使用 `log.error()`

## 2.2 日志规约

### 🔴【强制】使用 SLF4J 门面，不可直接使用 Log4j / Logback API

```java
// ✅ 正例（Lombok @Slf4j）
log.info("用户登录: userId={}", userId);

// ❌ 反例
System.out.println("xxx");  // 绝对禁止
```

### 🔴【强制】日志级别规范

| 级别 | 场景 | 本项环境 |
|------|------|:--:|
| **ERROR** | 系统错误需立即处理 | dev + prod |
| **WARN** | 业务异常可恢复 | dev + prod |
| **INFO** | 关键业务流程节点 | dev + prod |
| **DEBUG** | 调试信息 | dev only |
| **TRACE** | 极详细追踪 | 默认关闭 |

### 🔴【强制】敏感信息禁止明文打印

```java
// ❌ 反例：密码/Token 明文日志
log.info("登录: password={}", password);

// ✅ 正例：脱敏
log.info("登录: userId={}", user.getUserId());
```

### 🔴【强制】日志格式必须包含 traceId + spanId

```
%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level [%X{traceId},%X{spanId}] %logger{36} - %msg%n
```

### 🟡【推荐】Controller 日志使用统一前缀 `[{Module}]`

```java
log.info("[User] 注册: tenantId={}, username={}", ...);
log.info("[Auth] 登录成功: userId={}", ...);
log.info("[Role] 分配角色: roleId={}, userId={}", ...);
```

### 🟡【推荐】条件日志用占位符而非字符串拼接

```java
// ✅ 正例
log.debug("查询参数: id={}, name={}", id, name);

// ❌ 反例
log.debug("查询参数: id=" + id + ", name=" + name);  // 始终执行字符串拼接
```

# 三、MySQL 数据库规约

## 3.1 建表规约

### 🔴【强制】表名/字段名必须小写，下划线分隔

| ✅ 正例 | ❌ 反例 |
|---------|---------|
| `t_tenant`, `tenant_id`, `created_at` | `TTenant`, `tenantId`, `createdAt` |

### 🔴【强制】表名必须以 `t_` 开头，禁用数据库关键字

### 🔴【强制】所有业务表必须含以下三个字段

```sql
id          BIGINT       NOT NULL AUTO_INCREMENT  COMMENT '主键',
created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
```

### 🔴【强制】逻辑删除使用 `deleted` 字段（TINYINT，0=正常/1=删除）

### 🔴【强制】多租户表必须含 `tenant_id` 字段

```sql
tenant_id VARCHAR(64) NOT NULL COMMENT '租户标识'
```

### 🔴【强制】varchar 字段必须指定长度

```sql
-- ✅ 正例
username VARCHAR(64) NOT NULL COMMENT '用户名'

-- ❌ 反例
username VARCHAR NOT NULL
```

### 🔴【强制】小数类型使用 `DECIMAL`，禁止 `FLOAT` / `DOUBLE`

```sql
-- ✅ 正例：金额/比例等精确小数
price DECIMAL(10,2) NOT NULL COMMENT '价格'

-- ❌ 反例：存在精度丢失
price FLOAT NOT NULL
```

### 🔴【强制】字段注释必须完整，禁止无注释的字段

### 🟡【推荐】单表行数超过 500 万或容量超过 2GB 时分库分表

### 🟡【推荐】字段允许 NULL 时用 `NULL`，不允许时用 `NOT NULL DEFAULT ''`

## 3.2 索引规约

### 🔴【强制】业务上唯一特性的字段必须建唯一索引

### 🔴【强制】禁止超过 3 个表的 JOIN，需要 JOIN 时字段类型必须一致

> ⚠️ **本项目强制：禁止任何 JOIN，全部改为单表查询**

### 🔴【强制】varchar 字段建索引必须指定索引长度

```sql
INDEX idx_name (name(32))  -- 长字符串字段只索引前 32 位
```

### 🔴【强制】ORDER BY 字段尽量走索引，避免 filesort

### 🟡【推荐】利用覆盖索引避免回表查询

### 🟡【推荐】索引数量不超过 7 个

## 3.3 SQL 语句

### 🔴【强制】COUNT(*) / COUNT(1) 禁止使用 COUNT(列名)

### 🔴【强制】分页查询必须有排序条件，防止数据重复/丢失

```sql
SELECT * FROM t_user
WHERE tenant_id = #{tenantId} AND deleted = 0
ORDER BY id DESC  -- 必须有排序
LIMIT #{offset}, #{size}
```

### 🔴【强制】WHERE 条件中禁止函数运算（会导致索引失效）

```sql
-- ❌ 反例：索引失效
SELECT * FROM t_user WHERE DATE(created_at) = '2026-06-14'

-- ✅ 正例
SELECT * FROM t_user WHERE created_at >= '2026-06-14 00:00:00' AND created_at < '2026-06-15 00:00:00'
```

### 🔴【强制】禁止 SELECT *，必须列出具体字段

```sql
-- ✅ 正例
SELECT id, username, email, status FROM t_user WHERE tenant_id = #{tenantId}

-- ❌ 反例
SELECT * FROM t_user
```

### 🔴【强制】UPDATE 必须有 WHERE 条件（除非明确要对全表操作）

```sql
-- ❌ 反例：忘写条件导致全表更新
UPDATE t_user SET status = 'ACTIVE'

-- ✅ 正例
UPDATE t_user SET status = 'ACTIVE' WHERE id = #{id} AND deleted = 0
```

### 🔴【强制】数据订正前先 SELECT 验证影响行数再执行 UPDATE/DELETE

## 3.4 ORM 映射 + 项目单表查询铁律

### 🔴【强制】本项目强制：所有查询必须单表，禁用 JOIN

| ✅ 子查询（每层单表） | ❌ 多表 JOIN |
|---------|---------|
| `WHERE id IN (SELECT ... FROM table_a)` | `FROM a INNER JOIN b ON a.id = b.a_id` |
| Java 层多次调用 Repository 组合数据 | Mapper XML 中写 `LEFT JOIN` |

**关联数据获取标准模式 — 三层拆解:**

```java
// 场景：根据 userId 查用户 → 角色 → 权限（涉及 4 张表）

// 第 1 步：查 t_user_role 取 role_id 列表（单表）
List<Long> roleIds = roleRepository.findRoleIdsByUserId(userId);

// 第 2 步：查 t_role_permission 取 permission_id 列表（单表）
List<Long> permIds = new ArrayList<>();
for (Long roleId : roleIds) {
    permIds.addAll(permissionRepository.findPermIdsByRoleId(roleId));
}

// 第 3 步：查 t_permission 取权限详情（单表）
List<Permission> permissions = permissionRepository.findByIds(permIds);
```

```xml
<!-- ✅ MyBatis 子查询方式（每层只访问一张表） -->
<select id="findByUserId" resultMap="PermissionPOMap">
    SELECT DISTINCT * FROM t_permission
    WHERE id IN (
        SELECT permission_id FROM t_role_permission
        WHERE role_id IN (
            SELECT role_id FROM t_user_role WHERE user_id = #{userId}
        )
    ) AND deleted = 0
</select>
```

### 🔴【强制】禁止使用 MyBatis-Plus 的 QueryWrapper / LambdaQueryWrapper

**说明:** 所有 SQL 必须在 XML Mapper 中明确定义，保证 SQL 可见、可控、可 review。

### 🔴【强制】Mapper XML 中参数使用 `#{}` 而非 `${}`

```xml
<!-- ✅ 正例：预编译防注入 -->
WHERE tenant_id = #{tenantId}

<!-- ❌ 反例：字符串拼接有 SQL 注入风险（仅 orderBy 等动态字段可用） -->
WHERE tenant_id = '${tenantId}'
```

### 🔴【强制】禁止直接使用 Map 作为 Mapper 入参/返回值

### 🟡【推荐】避免使用 MyBatis 缓存，统一使用 Redis 外部缓存

# 四、安全规约

### 🔴【强制】用户敏感数据（密码/手机号/邮箱）禁止明文存储

```java
// 密码: BCrypt(strength=12)
// 手机号/邮箱: AES 加密存储
```

### 🔴【强制】密码强度策略

- 最少 8 位
- 至少包含大写字母、小写字母、数字、特殊字符中的 **3 种**
- 由 `PasswordService.validateComplexity()` 统一校验

### 🔴【强制】Token 安全策略

| 策略 | 说明 |
|------|------|
| AccessToken 有效期 | 1 小时，持续活跃自动续期 |
| RefreshToken 有效期 | 7 天，Rotation 策略（用后即焚，防重放） |
| 存储 | RefreshToken 存 Redis，AccessToken 由 Sa-Token 管理 |
| 日志 | 禁止打印 Token 值 |

### 🔴【强制】SQL 注入防护

- 所有 SQL 参数使用 `#{}`（预编译）
- 禁止拼接用户输入到 SQL 字符串
- 动态排序字段必须使用白名单校验

### 🔴【强制】XSS 防护：用户输入必须转义

### 🔴【强制】敏感操作的日志保留 6 个月以上

### 🔴【强制】权限控制

- 所有 `/api/v1/**` 默认需登录
- 细粒度权限使用 `@SaCheckPermission("resource:action")`
- 角色操作使用 `@SaCheckRole("TENANT_ADMIN")`
- 租户数据隔离由 `TenantSqlInterceptor` 自动注入

# 六、工程结构

## 6.1 DDD 六模块

```
com.example.agent
├── common          ← 共享内核：Result、Exception、枚举、常量
├── domain          ← 领域层：Entity、Repository 接口、DomainService 接口
├── application     ← 应用层：ApplicationService、DTO、Assembler
├── infrastructure  ← 基础设施：RepositoryImpl、PO、Mapper、Config、Filter、AOP
├── interfaces      ← 接口层：Controller、Request/Response DTO、ExceptionHandler
└── bootstrap       ← 启动层：SpringBootApplication、YAML、logback
```

## 6.2 依赖方向（严格单向，禁止反向）

```
interfaces ──→ application ──→ domain ←── infrastructure
     ↓                              ↓         ↓
     └────────── common ←───────────┴─────────┘
```

| 规则 | 说明 |
|------|------|
| **domain 不依赖任何层的框架/注解** | 纯 POJO + 接口，零 Spring 依赖 |
| **infrastructure 实现 domain 接口** | 依赖倒置 |
| **interfaces → application** | Controller 不应直接注入 Repository |
| **禁止循环依赖** | 各模块依赖单向流动 |

## 6.3 应用分层（阿里规范）

```
Controller 层    ← 接口层：参数校验、格式转换、调用 Service
  ↓
Service 层       ← 业务逻辑层：编排、事务管理
  ↓
Repository 层    ← 数据访问层：MyBatis Mapper 调用
  ↓
Database
```

- **禁止 Controller 直接调用 Repository / Mapper**
- **禁止 Service 层包含 SQL / HTTP / 文件 IO 细节**

# 九、工具与禁止事项

## 9.1 必须使用的技术栈

| 工具 | 用途 |
|------|------|
| **Lombok** | 用 `@Slf4j`, `@Getter`, `@Builder`, `@RequiredArgsConstructor` |
| **Sa-Token** | 认证鉴权，不用 Spring Security |
| **MyBatis + XML Mapper** | 数据库访问，不用 JPA / MyBatis-Plus Wrapper |
| **Flyway** | 数据库版本迁移 |
| **Sentinel** | 限流熔断 |
| **Hutool** | 通用工具类 |
| **SLF4J + Logback** | 日志框架 |
| **JUnit 5 + Mockito** | 测试框架 |

## 9.2 禁止事项 🚫

| ❌ 禁止 | 说明 |
|---------|------|
| JOIN 查询 | 本项目铁律，一律单表查询 |
| MyBatis-Plus QueryWrapper | SQL 必须在 XML 中明确定义 |
| Spring Data JPA | 使用 MyBatis |
| `System.out.println()` | 用 `log.info()` |
| `e.printStackTrace()` | 用 `log.error()` |
| `Executors` 创建线程池 | 用 `ThreadPoolExecutor` |
| SimpleDateFormat 作 static | 用 `DateTimeFormatter` |
| Controller 直接调用 Mapper | 必须通过 Repository |
| domain 层引入 Spring 注解 | 领域层保持纯净 |
| 循环依赖 | 严格单向依赖 |
| 拼音命名 / 中文注释 | 英文优先 |
| 魔法值硬编码 | 提取为常量或枚举 |
| 被注释掉的代码 | 直接删除（Git 可追溯） |
