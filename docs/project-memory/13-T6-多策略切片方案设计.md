## 关键决策
- **语义切片（semantic）与段落+滑动窗口严格区分**：原方案将段落切分称为"语义切分"名不副实，新方案中 semantic 是独立策略，基于 embedding 相似度检测断点
- **三层降级链**：手动指定 > 知识库默认 > 文件类型兜底，保证永远有合适的策略可用
- **动态升级规则**：PDF/DOCX 检测到标题自动升档为 markdown_header_aware；TXT 检测到 FAQ 模式自动升档为 sentence_level
- **每种策略有独立参数空间**：通过 chunk_config_json 灵活扩展，不互相污染
- **仅技术方案，不做代码开发**：本次只输出设计文档

## 踩坑记录
- 原方案 `02-语义切分向量化与Milvus.md` 中的 ChunkService 虽然方法名带 semantic，但实现是纯 `\n\n` 正则分割，无任何语义感知 → 新方案做了正名
- 原方案 chunk_size=512、overlap=50 全硬编码 → 新方案每种策略有独立的默认值和可配置参数
