## 关键决策

### 状态模型
- **KnowledgeBaseStatus**: ENABLED → DISABLED ↔ ENABLED（可逆）→ DELETED（终态）
- **DocumentStatus 重写**: UPLOADING→PENDING_PARSE, READY→PARSED, 新增 DEPRECATED
- **二步法安全删除**: PARSED 文档必须先弃用(deprecate)再删除(delete)，防止误删
- **KB 删除二步法**: ENABLED → DISABLED → DELETED，防止误删

### 创建者权限（需求 7）
- KB 新增 `createdBy` 字段，删除 KB/文档时检查 `kb.createdBy == currentUserId`
- 权限检查在 `KnowledgeBaseDomainService.assertCreatorAccess()` 统一执行
- `kb:admin` 权限可绕过创建者限制

### 检索过滤修复（需求 6）
- `HybridSearchApplicationService.search()` 的 `knowledgeId` 参数当前被忽略 → 需修复
- Milvus collection 新增 `knowledge_id` 字段 → `VectorEntry` 携带 `knowledgeId`
- `VectorSearchProvider.search()` 新增 `filterExpression` 参数
- `FulltextSearchProvider.search()` 改为接收 `Set<String> knowledgeIds`

### 检索结果溯源（需求 8）
- `SearchResultDTO.HitItem` 已有 `documentFilename` + `documentAccessUrl` ✅
- `SearchResultDTO.DocumentRef` 已有 `filename` + `accessUrl` ✅
- 需放宽下载 Controller：允许 PENDING_PARSE / PARSED / DEPRECATED 状态下载

## 数据库变更（V1.4.0）

### t_knowledge_base
- 新增 `created_by VARCHAR(64) NOT NULL`（创建者追踪）
- 数据迁移：`status='ACTIVE'` → `'ENABLED'`

### t_document
- 数据迁移：`status='UPLOADING'` → `'PENDING_PARSE'`, `status='READY'` → `'PARSED'`
- 新增联合索引：`idx_doc_kb_status (knowledge_id, status)`

### t_document_chunk
- 新增 `deleted TINYINT DEFAULT 0`（软删标记，弃用时标记而非物理删除）

### Milvus Collection Schema
- 每个 tenant collection 新增 `knowledge_id VARCHAR(128)` 字段
