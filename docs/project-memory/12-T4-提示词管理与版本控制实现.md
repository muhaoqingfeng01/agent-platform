## 关键设计决策

1. **VariableResolver Chain** — 变量解析采用 Strategy + Chain of Responsibility 双模式：
   - 每个 Resolver 声明 `priority()` 决定链中顺序
   - 系统变量（10）→ 上下文变量（20）→ 兜底（100）
   - 新增变量来源只需实现接口 + `@Component`，自动注入链

2. **Memento 模式用于版本管理** — 发布/回滚每次创建 `PromptTemplateVersion` 快照，回滚是新版本（非覆盖），保留完整审计链

3. **State 模式用于状态流转** — `PromptStatus` 枚举自带 `allowedTransitions()`，非法状态转移在领域层即被拦截

4. **DTO 分离遵循 DDD 约束** — Application 层定义响应 DTO（内部类），Interfaces 层定义请求 DTO，Controller 做转换

5. **JsonUtil 未抽取** — 当前各 RepositoryImpl 各自持有 `ObjectMapper` 实例（与 IntentRepositoryImpl 一致），后续可统一抽取到 common 模块

## 踩坑记录
