# 会话 04 — 开发计划与技术方案拆分

**日期**: 2026-06-11
**触发**: 需要分析功能优先级、细化每个功能的实现内容、拆分技术文档

---

## 执行的操作

### 1. 制定 P0-P4 开发优先级

```
P0 (5天)    P1 (8天)     P2 (6天)     P3 (3天)     P4 (4天)
基础底座  → 核心能力   → 增强能力  → 安全治理  → 观测优化
T2 + T8     T3+T4+T5     T6 + T7     T10 + T11    T9 + T12
```

### 2. 细化每个功能实现清单

**文件**: `docs/后端开发计划.md`

每个阶段包含：
- 功能点序号和名称
- 具体实现内容描述
- 涉及文件路径
- API 端点设计
- 涉及的数据库表
- Swagger 标签
- 验证方式

共梳理出 **20 个 Controller**、**60+ 个 API 端点**。

### 3. 技术方案文档拆分

将 12 个阶段（T1-T12，后整合为 P0-P5）拆分为 **28 个子项详细技术方案**，按 P0-P4 优先级组织。

每个子方案包含：
- 使用技术栈
- 涉及数据库表
- API 端点列表
- 核心代码示例（Java）
- 实现细节说明
- 验证方式

### 4. 同时保留原始文档

原有的 12 个 T1-T12 汇总方案（如 `T1-交互端构建/技术方案.md`）保持不变，作为概览参考。

> **2026-06-13 更新**: T1-T12 目录已删除，全部技术方案已拆分至 P0-P5 目录下，T1 对应 P5-交互端。
