# 07-日志链路追踪与数据库仓储实战化

> **日期**: 2026-06-14 13:11
> **分支**: master
> **触发**: 代码生成（新增 12 文件，修改 10 文件）

## 做了什么

### 1. 全链路追踪 TraceFilter
- 新增 `TraceFilter.java` — Servlet Filter，优先级 `HIGHEST_PRECEDENCE`
- 每个请求自动生成 `traceId`（32位UUID）和 `spanId`（16位UUID）
- 支持上游传入 `X-Trace-Id` 请求头（跨服务传播）
- traceId 写入响应头 `X-Trace-Id`，前端可关联
- 请求完成后清理 MDC，防止 ThreadLocal 泄漏
- 日志模式从 `[%X{traceId}]` 升级为 `[%X{traceId},%X{spanId}]`

### 2. 数据库仓储全部实战化（MyBatis）
**新增 PO 类 3 个**:
- `TenantPO` — 映射 `t_tenant` 表
- `RolePO` — 映射 `t_role` 表
- `PermissionPO` — 映射 `t_permission` 表

**新增 Mapper 接口 3 个**:
- `TenantMapper` — 租户 CRUD
- `RoleMapper` — 角色 CRUD + 用户-角色关联 + 角色-权限关联
- `PermissionMapper` — 权限 CRUD + 按用户ID三表JOIN查询

**新增 XML 映射 3 个**:
- `TenantMapper.xml`
- `RoleMapper.xml`
- `PermissionMapper.xml`

**新增 Repository 实现 3 个** (全部 `@Primary`):
- `TenantRepositoryImpl` — 替换 `TenantRepositoryStub`
- `RoleRepositoryImpl` — 替换 `RoleRepositoryStub`
- `PermissionRepositoryImpl` — 替换 `PermissionRepositoryStub`

### 3. 服务层实战化改造
- `RoleServiceImpl.getRoleCodes()` → 从 DB 查询 `t_role JOIN t_user_role`
- `PermissionServiceImpl.getPermissionCodes()` → 从 DB 三表 JOIN 查询权限码
- `UserServiceImpl.getRoleCodes()` → 委托给 `RoleService`（不再硬编码）
- `UserServiceImpl.authenticate()` → 保留 admin mock 流程 + **标记 TODO**
- `PermissionRepository` 接口新增 `findByUserId()` 方法

### 4. Stub 类废弃
- 三个 Stub 全部 `@Deprecated`，`@Repository` 注释掉
- Spring 自动注入 `@Primary` 的实际实现

## 关键决策
- **TraceFilter 用 Servlet Filter 而非 Interceptor** — Filter 更底层，确保 MDC 在所有组件之前设置
- **spanId 始终新建** — 每次服务调用生成新 spanId，traceId 可跨服务传播
- **Repository 接口不添加 `findByUserId` 给 Tenant** — 无此业务需求
- **admin mock 暂不删除** — 标记 TODO，等待 P0 完成后统一处理

## 踩坑记录
- **PermissionRepository 接口新增方法后，Stub 也必须同步添加** — 否则编译失败
- **Spring Bean 冲突**: `@Primary` impl + `@Repository` stub = 启动报错，必须注释掉 stub 的 `@Repository`
- **Stub 的 `save()` 回填 ID 不生效** — 原代码 `tenant = saved` 在 Java 中是引用重赋值，不影响调用方，这是预存问题
