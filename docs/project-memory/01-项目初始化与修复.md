# 会话 01 — 项目初始化与修复

**日期**: 2026-06-10 ~ 2026-06-11
**触发**: 项目无法正常启动，需要检查问题并修复

---

## 原始问题

1. 项目无法编译启动
2. JDK 版本不匹配（要求 21，系统只有 17）
3. 存在大量无用的 Java 空代码

---

## 执行的操作

### 1. JDK 降级 21 → 17

**修改文件**: `pom.xml`（父 POM）

```xml
<java.version>17</java.version>
<maven.compiler.source>17</maven.compiler.source>
<maven.compiler.target>17</maven.compiler.target>
```

### 2. 依赖版本同步降级

| 依赖 | 原版本 → 新版本 | 原因 |
|------|----------------|------|
| spring-boot | 3.5.13 → **3.3.7** | JDK 17 最佳兼容，匹配 Spring Cloud 2023.0.x |
| micrometer | 1.14.5 → **1.13.8** | 匹配 Spring Boot 3.3.x |
| maven-compiler-plugin | 3.13.0 → **3.12.1** | 降低兼容性风险 |
| maven-surefire-plugin | 3.5.2 → **3.2.5** | 降低兼容性风险 |

### 3. 修复依赖坐标错误

**Group ID 修正**:
- `com.alibaba.cloud` → `com.alibaba.cloud.ai`（Spring AI Alibaba 的 groupId 含 `.ai` 后缀）

**Artifact ID 修正**:
- `spring-ai-alibaba-starter` → `spring-ai-alibaba-agent-framework`（新版模块化）
- `spring-ai-mcp-client-spring-boot-starter` → `spring-ai-starter-mcp-client`
- `spring-ai-rag-core` → `spring-ai-rag`

### 4. 修复不存在的版本号

| 依赖 | 错误版本 → 正确版本 |
|------|-------------------|
| `mysql-connector-j` | 3.0.33 → **8.0.33** |
| `langfuse-java` | 2.0.4 → **0.2.0** |

### 5. 新增缺失的 BOM

添加 `org.springframework.ai:spring-ai-bom:1.1.7` 管理 Spring AI 依赖版本。

### 6. 清理无用代码

- 删除 `AgentPlatformConfig.java`（空 `@Configuration`，无任何 Bean）
- 删除 **100+ 个空目录**（`model/`、`service/`、`repository/`、`event/`、`dto/` 等）

---

## 最终结果

- ✅ `mvn compile` **BUILD SUCCESS**（7/7 模块）
- ✅ 使用 `javac debug target 17` 编译
- ✅ 保留 18 个 Java 文件（1 主类 + 1 测试 + 16 package-info）

---

## 关键决策

1. **Spring Boot 3.3.7 而非 3.4.x/3.5.x** — 3.3.x 是最广泛使用的稳定版，与 Spring Cloud 2023.0.x 完美匹配
2. **Spring AI Alibaba 保持 1.1.2.0** — 最新稳定版，仅修正了 groupId
3. **保留 package-info.java** — 文档价值大于清理价值
4. **删除 100+ 空目录** — 防止误导，明确哪些包有代码、哪些没有
