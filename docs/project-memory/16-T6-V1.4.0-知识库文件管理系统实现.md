## 关键决策
- **二步法安全删除**: PARSED 文档必须先 deprecate（删向量）再 delete；ENABLED KB 必须先 disable 再 delete
- **检索过滤三级策略**: 指定 KB → 单 KB filter；未指定 → 所有 ENABLED KB filter；无 KB → null（搜全部）
- **下载权限放宽**: PENDING_PARSE/PARSED/DEPRECATED 均可下载（文件在 MinIO），仅 DELETED 不可
- **DocumentStatus 枚举重写**: 旧值 UPLOADING→PENDING_PARSE、READY→PARSED，新增 DEPRECATED
- **Milvus 标量索引**: 为 document_id 和 knowledge_id 创建 TRIE 索引，支持精确匹配过滤
