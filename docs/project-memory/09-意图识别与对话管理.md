## 做了什么

### P1-T3 意图识别与对话管理 — 核心实现 (~75%)

**意图识别引擎（责任链模式，3 层串联）:**
- `LLMRecognizer` — LLM 兜底分类器（Template Method 模式），通过 `ChatClient` 调用 DeepSeek，JSON 结构化输出
- `RuleRecognizer` — 规则优先匹配（关键词/模式），~1ms 级响应
- `CacheRecognizer` — Redis 缓存命中（TTL 30min），~5ms 级响应
- `IntentRecognitionChain` — 责任链编排：规则(≥0.9) → 缓存 → LLM 兜底，命中即短路
- `IntentApplicationService` — CRUD + 测试识别 + 批量测试
- `IntentController` — 7 个 REST 端点（CRUD、状态切换、单条测试、批量测试）

**对话会话管理:**
- `ConversationApplicationService` — 会话 CRUD + 状态流转
- `MessageApplicationService` — 消息 CRUD + 分页查询
- `ConversationController` + `MessageController` — REST 端点
- `ConversationDomainService` — 领域服务（会话生命周期校验）
- 领域事件：`ConversationCreatedEvent`、`ConversationStatusChangedEvent`

**流式编排（SSE + WebSocket）:**
- `StreamOrchestrationService` — 流式对话编排（责任链 + Observer 模式）
- `SseEmitterFactory` / `SseEventFactory` — SSE 基础设施
- `ConversationWebSocketHandler` / `WebSocketConfig` — WebSocket 通道
- `StreamThreadPoolConfig` — 流式处理专用线程池

**会话状态机:**
- `ConversationStateMachineConfig` — Spring StateMachine 4.0.0 配置
- `ConversationState` / `ConversationEvent` — 状态与事件枚举

**记忆管理（Template Method + Factory 模式）:**
- `LongTermMemoryService` — 长期记忆提取与检索（异步 @Async）
- `SessionMemoryService` — Redis 会话上下文管理
- 5 个 `MemoryExtractor`：Context / Fact / Preference / Summary + Registry

**Domain 层:**
- 4 个实体：`Conversation`、`Intent`、`LongTermMemory`、`Message`
- 4 个仓储接口 + 4 个仓储实现（MyBatis）
- 6 个值对象：`ConversationStatus`、`FeedbackType`、`IntentCategory`、`IntentStatus`、`MemoryType`、`MessageRole`

**Infrastructure 层:**
- `AiConfig` — ChatClient Bean 定义（解决 Spring AI 只自动配置 Builder 不配置 Client 的问题）
- 4 个 PO（`ConversationPO`、`IntentPO`、`LongTermMemoryPO`、`MessagePO`）
- 4 个 Mapper 接口 + 4 个 MyBatis XML
- `Flyway V1.2.1` — 业务 ID 字段补充

**Common 层:**
- `PageResponse` — 通用分页响应
- `IdGenerator` — 业务 ID 生成器

## 关键决策

1. **LLM 选型**: 从 DashScope 切换到 DeepSeek — Spring AI Alibaba 1.1.2.0 BOM 中无 DashScope 自动配置，仅 `spring-ai-deepseek` 可用
2. **ChatClient bean**: Spring AI 只自动配置 `ChatClient.Builder`，需手动 `@Bean ChatClient`（`AiConfig.java`）
3. **自动配置依赖**: 必须显式添加 `spring-ai-autoconfigure-model-deepseek` + `spring-ai-autoconfigure-model-chat-client`，否则 ChatModel/ChatClient.Builder 均不会自动创建
4. **意图识别三层架构**: Rule(~1ms) → Cache(~5ms) → LLM(~500ms)，命中即短路，兼顾性能与准确率
5. **记忆提取异步化**: LongTermMemoryService 使用 `@Async`，避免阻塞对话主流程

## 踩坑记录

1. **`spring-ai-alibaba-agent-framework` 无 ChatModel**: 只提供 Agent 编排，ChatModel 需单独引入
2. **`mvn compile` ≠ `mvn install`**: 新增依赖后必须 install 到本地仓库，bootstrap 模块才能解析传递依赖
3. **Spring AI 1.1.7 无 `ChatClient` bean**: 只有 `ChatClient.Builder`，需手动包装
