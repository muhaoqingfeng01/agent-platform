## 关键决策
- **IVF_SQ8 是中后期推荐甜点位**：存储减少 75%，精度损失 < 0.5%，90% 生产 RAG 系统的选择
- **HNSW 为实时对话场景首选**：检索延迟 < 5ms，用内存换低延迟
- **nprobe = nlist × 0.1 作为默认值**：当前硬编码 nlist=128 但 nprobe 未设，需立即补充
- **相似度阈值 0.50 作为平衡模式默认**：过滤低质量结果，降低 RRF 融合噪声
- **不推荐 P2 初期引入多阶段检索**：优先补齐基础参数和评估数据集，Reranker 在 Phase 3 引入

## 踩坑记录
- 原方案 02 中 nlist=128 但没有 nprobe → Milvus 默认 nprobe≈8，预期 recall@10 仅 ~0.88，生产不可接受
- 原方案 03 中向量检索结果不经过滤直接送 RRF → 低相似度噪声可能污染融合排序
