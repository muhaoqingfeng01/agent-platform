> **日期**: 2026-06-18
> **分支**: master

## 做了什么

按 `docs/P6-迭代优化方案/P0-P3-未完成事项-迭代方案/` 方案实现了全部 9 项功能：

### 第一梯队 P1（检索与工具质量）
- **Reranker 重排序**: `Reranker` 接口 + `LLMReranker` + `CrossEncoderReranker`(stub) + `RerankerRegistry` → 集成到 `HybridSearchApplicationService.search()`
- **工具版本化**: Flyway V1.5.0 建表 + `ToolRegistryVersion` 实体/仓储/PO/Mapper + `ToolDomainService.rollback()` + API 端点

### 第二梯队 P2（性能优化）
- **Redis 提示词缓存**: `CachedPromptRepository`(装饰器) + `CacheConfig` → 发布/回滚/归档时自动更新/清除缓存
- **Redis 工具缓存**: `ToolCacheManager`(L2 Redis) → 集成到 `ToolApplicationService.getByToolId()`
- **精度监控闭环**: `RetrievalPrecisionService`(recall@5/MRR/NDCG@5/HitRate) + `GridSearchOptimizer` + `PrecisionController`

### 第三梯队 P3（运维安全增强）
- **MCP 心跳检测**: `McpHeartbeatDetector`(30s间隔，连续3次失败自动禁用) + `McpClientManager.ping()/removeClient()`
- **LDAP/SSO 企业认证**: `AuthProviderType` + `LdapAuthenticationProvider` + `AuthProviderFactory` + `AuthController` 扩展
- **Presidio PII 增强**: `PresidioPiiAdapter` + `PiiDesensitizer` 增强(合并去重+降级)

## 关键决策
- Reranker 统一接口 `domain.knowledge.service.Reranker`，LLM 实现在 application 层，CrossEncoder 在 infrastructure 层
- 缓存管理在 Application 层调用（避免 Domain 依赖 Infrastructure）
- `ToolRegistry` 增加 `@Setter` 和 `version` 字段以支持版本化
- 工具版本化使用独立的 `t_tool_registry_version` 表，遵循 Memento 模式
- Presidio 条件装配，不可用时静默降级到正则
