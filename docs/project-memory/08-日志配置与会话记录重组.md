## 关键决策
- Logback 日志活跃文件不指定 `<file>`，从 `fileNamePattern` 推导，实现活跃日志直接写入日期目录
- Session 快照与项目文档分离 — `sessions/` 独立目录存放自动生成的快照，`project-memory/` 根目录存放人工编写的项目记忆
- 日志级别通过 `logback-spring.xml` 的 `<springProfile>` 统一管理，不再分散在 YAML 中

## 踩坑记录
- `SizeAndTimeBasedRollingPolicy` 如果指定了 `<file>` 为静态路径，当天活跃日志会写在根目录而非日期目录。去掉 `<file>` 后 Logback 从 `fileNamePattern` 自动推导，活跃日志直接进入日期子目录
- `fileNamePattern` 中的 `%d` 参数既是时间滚动边界（每天 00:00 触发），也是归档路径模板
- `.session-log.jsonl` 是隐藏文件（`.` 开头），`ls` 不显示，需用 `ls -la`
