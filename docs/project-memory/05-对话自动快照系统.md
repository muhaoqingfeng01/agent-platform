## 做了什么
- 创建 `.claude/hooks/session-snapshot.py` — git 元数据捕获脚本（**仅在有变更时记录**）
- 创建 `.claude/settings.json` — 项目级 Stop Hook
- 更新 `CLAUDE.md` — 添加"对话快照规则"章节
- **精简设计**: 移除 PostToolUse hook，只在 Stop 时检测变更，无变更静默跳过

## 关键决策
- **两层设计**（精简后）:
  1. **AI 层**（CLAUDE.md 指令）→ 我主动写 markdown 快照（有智能）
  2. **Hook 层**（Stop Hook）→ 仅当 git 有变更时写 JSON 日志（无智能但可靠）
- 去掉了 PostToolUse hook（每次 Edit/Write 都触发太频繁，且大部分中间状态无意义）
- **变更检测前置**: 脚本启动先 `git diff --name-only` + `git ls-files --others`，全部为空则直接 `return`，不写任何文件

## 踩坑记录
- 全局 hooks.json 已有 `lessons-extractor`，但它写的是全局 memory 系统（`~/.claude/projects/.../memory/`），不写项目目录
- 项目级 hook 不能直接访问 LLM → 脚本只能捕获元数据，语义总结靠 CLAUDE.md 指令驱动
